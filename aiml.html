AI vs Machine Learning — What's the Difference?

These terms are often used interchangeably but they are not the same thing. Understanding the distinction prevents confusion when choosing tools or reading research.

Artificial Intelligence (AI)

The broadest term. Any technique that enables machines to simulate human-like intelligence — reasoning, problem-solving, planning, language understanding, perception.

Includes:

  • Rule-based expert systems (no learning)
  • Search algorithms (A*, minimax)
  • Machine Learning (learns from data)
  • Robotics and planning
  • Natural language processing
AI does not require learning from data. A chess engine with hard-coded rules is AI but not ML.
Machine Learning (ML)

A subset of AI where systems learn patterns from data without being explicitly programmed for every case. The model improves as it sees more examples.

Core idea:

Given input X and output Y, find a function f such that f(X) ≈ Y — and generalise to unseen X.

Three families:

  • Regression — predict a continuous value
  • Classification — predict a category
  • Clustering / Generation — discover structure
Deep Learning (DL)

A subset of ML using multi-layer neural networks with many parameters (millions to billions). Excels on unstructured data: images, audio, text, video.

Why "deep"?

Multiple stacked layers learn increasingly abstract representations — edges → shapes → objects → concepts.

When to use DL over classical ML:

  • Input is images, audio, or raw text
  • You have >100k training examples
  • Feature engineering would be impractical
  • State-of-the-art accuracy required

Types of Machine Learning

Categorised by how the model learns from data.

Supervised Labelled data

Every training example has an input X and a known correct output y. The model learns to map X → y and is evaluated on how well it predicts y on unseen data.

Use when: You have labelled historical data and want to predict a target.

ClassificationRegressionObject detectionTranslation
Unsupervised No labels

Only inputs X, no labels. The model discovers hidden structure — clusters, patterns, compressed representations — purely from the distribution of the data.

Use when: You have lots of data but no labels, or want to explore structure.

ClusteringDimensionality reductionAnomaly detection
Reinforcement Reward signal

An agent takes actions in an environment, receives a reward, and learns a policy that maximises cumulative reward over time. No labelled data — the signal comes from outcomes.

Use when: Sequential decision-making with a clear reward structure.

GamesRoboticsTradingAd bidding
Self-Supervised Labels from data itself

A special form of supervised learning where labels are generated from the input data itself. The model learns by predicting masked/hidden parts of inputs. Powers modern LLMs.

Use when: You have massive unlabelled data and want rich representations.

A small labelled set + a large unlabelled set. The model uses the labelled data for direct supervision and the unlabelled data to learn better representations. Practical when labelling is expensive.

Take a model pretrained on a large general dataset (ImageNet, Common Crawl) and fine-tune it on your smaller task-specific dataset. Dramatically reduces data and compute requirements.

Random Forest Intermediate

An ensemble of decision trees, each trained on a random subset of data and features. Final prediction = majority vote (classification) or mean (regression). Resistant to overfitting; handles missing values.

Key hyperparameters:

  • n_estimators — number of trees (more = better, diminishing returns)
  • max_depth — controls individual tree complexity
  • max_features — features sampled per split (√n for classification)
  • min_samples_split — prevents overfitting on small nodes
When to use: Default choice for tabular data before trying anything more complex.

Gradient boosting builds trees sequentially — each tree corrects the errors of the previous one. Dominates Kaggle competitions on structured data. Faster and more accurate than Random Forest on most tabular problems.

  • learning_rate — step size (lower = more trees needed)
  • n_estimators + early_stopping_rounds
  • max_depth / num_leaves — tree complexity
  • subsample, colsample_bytree — regularization via sampling
Rule: Low learning rate (0.01–0.05) + many trees + early stopping = gold standard.

Regression — From Linear to Neural

Regression predicts a continuous numeric output (price, temperature, score). Understanding the regression family in depth is the foundation of almost all supervised learning, including neural networks.

Linear RegressionBeginner

Fits a straight line (or hyperplane) through the data: ŷ = w₀ + w₁x₁ + w₂x₂ + … + wₙxₙ. Weights are found by minimising the Mean Squared Error (MSE) between predictions and true values.

from sklearn.linear_model import LinearRegression model = LinearRegression() model.fit(X_train, y_train) print(model.coef_) # weight per feature print(model.intercept_) # bias term w₀

Assumptions (when they break, the model fails):

  • Linear relationship between features and target
  • Errors are normally distributed and homoscedastic
  • Features are not perfectly collinear
Ridge & Lasso (Regularised Regression)Beginner

Plain linear regression overfits with many features. Regularisation adds a penalty to large weights, forcing the model to stay simpler.

MethodPenaltyEffect
Ridge (L2)λ · Σwᵢ²Shrinks all weights toward zero; keeps all features
Lasso (L1)λ · Σ|wᵢ|Drives some weights to exactly zero — automatic feature selection
ElasticNetmix of L1+L2Best of both; use when features are correlated
from sklearn.linear_model import Ridge, Lasso, ElasticNet ridge = Ridge(alpha=1.0) # alpha = λ lasso = Lasso(alpha=0.1) enet = ElasticNet(alpha=0.1, l1_ratio=0.5)
Polynomial RegressionBeginner

Fits curves instead of lines by adding polynomial features: x², x³, x·z, etc. Still linear in the weights — just linear regression on transformed features.

from sklearn.preprocessing import PolynomialFeatures from sklearn.pipeline import make_pipeline model = make_pipeline( PolynomialFeatures(degree=3), Ridge(alpha=1.0) # always regularise poly ) model.fit(X_train, y_train)
Degree > 3 almost always overfits badly. Prefer tree-based models or neural networks for complex non-linear relationships.
Logistic RegressionBeginner

Despite the name, logistic regression is a classification model. It applies the sigmoid function to a linear combination of features to output a probability between 0 and 1.

ŷ = σ(w·x + b) where σ(z) = 1/(1 + e⁻ᶻ)

Threshold at 0.5 for binary classification. For multi-class, use softmax (multinomial logistic regression).

  • Fast to train, highly interpretable (odds ratios)
  • Great baseline for any classification problem
  • Fails on non-linear decision boundaries without feature engineering
Regression with Neural NetworksIntermediate

For complex non-linear regression, an MLP replaces the linear model: multiple hidden layers with non-linear activations, then a single linear output neuron (no activation on the last layer for regression).

class MLPRegressor(nn.Module): def __init__(self): super().__init__() self.net = nn.Sequential( nn.Linear(n_features, 128), nn.ReLU(), nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, 1) # linear output ) def forward(self, x): return self.net(x)

Loss: nn.MSELoss() for regression. Use nn.HuberLoss() for robustness to outliers.

Unsupervised Learning Models

Finding structure without labels.

K-Means Clustering
Assigns each point to the nearest of K centroids, then recalculates centroids iteratively. Fast, simple, scales well. Assumes spherical clusters of similar size.

Choose K: Elbow method (plot inertia vs K) or Silhouette score.

Density-based clustering. Groups points that are closely packed; marks low-density points as noise/outliers. Doesn't require specifying K; discovers arbitrary-shaped clusters.

Linear dimensionality reduction. Projects data onto orthogonal axes of maximum variance. Use to reduce features, visualise high-dim data, or remove noise before another model.

Non-linear dimensionality reduction for visualisation only (2D/3D plots). t-SNE preserves local structure; UMAP is faster and better preserves global structure.

Deep Learning Architectures

Neural network families — when to use which.

Neural Network Fundamentals — The MLP Foundation

A Multi-Layer Perceptron (MLP) — also called a fully-connected or feed-forward network — is the simplest neural network. Every advanced architecture (CNN, RNN, Transformer) is built on the same core idea.

Anatomy of a neuron:

output = activation(Σ(weightᵢ × inputᵢ) + bias)

Anatomy of a network:

  • Input layer — one node per feature
  • Hidden layers — learn non-linear representations
  • Output layer — one node per class (classification) or 1 node (regression)

How it learns:

  1. Forward pass: input flows through layers, produces prediction
  2. Loss: compare prediction to truth (MSE, cross-entropy)
  3. Backward pass: gradients flow back via chain rule (backprop)
  4. Update: optimizer adjusts weights to reduce loss
  5. Repeat for every batch until loss converges
import torch.nn as nn # 3-layer MLP: 784 → 256 → 128 → 10 # (e.g., MNIST: 28×28 pixels → 10 digit classes) model = nn.Sequential( nn.Linear(784, 256), # hidden layer 1 nn.ReLU(), nn.Dropout(0.3), nn.Linear(256, 128), # hidden layer 2 nn.ReLU(), nn.Dropout(0.3), nn.Linear(128, 10), # output (logits) ) # Training step logits = model(x_batch) # forward loss = nn.CrossEntropyLoss()(logits, y_batch) loss.backward() # backward optimizer.step() # update

Key design choices:

ChoiceDefault Recommendation
Activation (hidden)ReLU (or GELU for transformers)
Activation (output)None (regression), Softmax (classification)
Width128–512, powers of 2 for GPU efficiency
DepthStart with 2–3 hidden layers

CNN — Convolutional Neural Network Intermediate

Applies learned filters (kernels) across input to detect spatial patterns — edges, textures, shapes. Translation-invariant. Standard architecture: Conv layers → ReLU → Pooling → Fully connected.

Key layers: Conv2D, MaxPool, BatchNorm, GlobalAvgPool, Dense

Use for: Images, video frames, audio spectrograms, spatial data

Notable models: ResNet, EfficientNet, VGG, MobileNet, YOLO (detection)

RNN / LSTM / GRU Intermediate

Recurrent networks process sequences by maintaining a hidden state across timesteps. LSTM adds gates (forget/input/output) to control long-range memory. GRU is a lighter variant.

Problem: Vanishing gradients over long sequences — largely replaced by Transformers.

Use for: Time series, legacy NLP, short sequences

Still useful when: Low compute, real-time streaming, short context

Transformer Advanced

Built on self-attention — every token attends to every other token in parallel. No recurrence means it scales with compute. The dominant architecture for language, code, images (ViT), and multimodal models.

Core concepts: Query/Key/Value attention, multi-head attention, positional encoding, feed-forward sublayers, layer norm

Notable models: GPT, BERT, T5, LLaMA, Mistral, Vision Transformer (ViT)

GAN — Generative Adversarial Network Advanced

Two networks in competition: a Generator that creates fake samples and a Discriminator that distinguishes real from fake. Generator improves until the discriminator can't tell the difference.

Training challenge: Mode collapse, training instability, GAN game doesn't always converge

Notable: StyleGAN (faces), CycleGAN (image translation), Pix2Pix

Diffusion Model Advanced

Learns to reverse a noise-adding process. Training: gradually add Gaussian noise to data. Inference: start from pure noise and denoise iteratively. Produces higher quality and more stable output than GANs.

Key concept: DDPM (Denoising Diffusion Probabilistic Model), U-Net backbone, classifier-free guidance

Notable: Stable Diffusion, DALL-E 3, Sora (video), AudioLDM

Graph Neural Network (GNN) Advanced

Operates directly on graph-structured data. Each node aggregates messages from its neighbours, updating its representation. Works on irregular, relational data that CNNs can't handle.

Variants: GCN, GraphSAGE, GAT (attention-based), MPNN

Use for: Social networks, molecules, knowledge graphs, recommendation systems

Reinforcement Learning

Training agents through trial and error in an environment.

Core components: Agent (learner) · Environment (world) · State (current situation) · Action (agent's choice) · Reward (signal) · Policy (action-selection strategy) · Value function (expected future reward)
Q-Learning / DQN Intermediate
Learns a Q-value (expected reward) for each state-action pair. DQN uses a neural network as the Q-function approximator, enabling it to handle high-dimensional state spaces (pixels).
Discrete actionsAtari games
Policy Gradient (REINFORCE) Advanced
Directly optimises the policy (probability distribution over actions) by gradient ascent on expected reward. High variance — needs many samples. Foundation of PPO and modern RLHF.
Continuous actionsLLM fine-tuning (RLHF)
PPO / Actor-Critic Advanced
Proximal Policy Optimization — constrains policy updates to a trust region for stability. Actor-Critic combines value estimation with policy learning. Current standard for RL in practice and RLHF.

Math & Statistics You Actually Need

You don't need a PhD — but these fundamentals are unavoidable.

📐 Linear Algebra

Data is tensors (scalars, vectors, matrices, higher-dim). All neural network operations are matrix multiplications.

  • Vectors: Feature representations, embeddings
  • Matrix multiply: Core of every dense layer (y = Wx + b)
  • Dot product: Similarity between embeddings (cosine similarity)
  • Eigenvalues/vectors: PCA, understanding variance
  • Transpose, inverse: Used throughout gradient derivations
Priority: Matrix multiply > dot product > eigenvalues. You can go far with just the first two.
📉 Calculus & Gradients

Backpropagation is just the chain rule applied repeatedly through a computation graph.

  • Derivative: Rate of change — how much loss changes with each weight
  • Partial derivative: Derivative w.r.t. one variable, holding others fixed
  • Gradient: Vector of all partial derivatives → direction of steepest ascent
  • Chain rule: ∂L/∂w = (∂L/∂a)(∂a/∂w) — propagates error backward
  • Jacobian/Hessian: Higher-order, used in advanced optimizers
Key insight: You don't derive gradients manually — frameworks (PyTorch autograd) do it. But understanding what's happening lets you debug and design better.
🎲 Probability & Statistics
  • Mean, variance, std dev: Describe and normalize your data
  • Distributions: Normal (Gaussian), Bernoulli, Categorical, Beta — model assumptions
  • Bayes' theorem: P(A|B) = P(B|A)P(A)/P(B) — foundation of probabilistic ML
  • MLE vs MAP: Maximum Likelihood Estimation vs Maximum A Posteriori — how models are trained
  • Entropy / KL divergence: Information content, used in cross-entropy loss and VAEs
  • Central Limit Theorem: Why sample means are normally distributed at scale

Training Pipeline

From raw data to a trained model — every step matters.

1 — Data Preparation

Split your data first — before any preprocessing:

  • Train (70–80%): Model learns from this
  • Validation (10–15%): Tune hyperparameters on this
  • Test (10–15%): Final honest evaluation — touch once
Never fit scalers/encoders on validation or test data — only fit on training, then transform all sets. Fitting on all data = data leakage.

Preprocessing steps:

  • Handle missing values (impute mean/median/mode or model-based)
  • Encode categoricals (one-hot, ordinal, target encoding)
  • Scale numerics (StandardScaler for NNs, MinMax optional)
  • Feature engineering (interactions, log transforms, rolling stats)
2 — Forward & Backward Pass

One training step:

  1. Forward pass: Input flows through the network → produces prediction ŷ
  2. Compute loss: Loss(ŷ, y) — how wrong was the prediction
  3. Backward pass: Chain rule propagates gradients from loss back through every layer
  4. Update weights: w ← w − lr × ∂L/∂w
  5. Zero gradients: Reset before next batch (PyTorch doesn't auto-zero)
for epoch in range(num_epochs): for X, y in train_loader: optimizer.zero_grad() # ① reset pred = model(X) # ② forward loss = criterion(pred, y) # ③ loss loss.backward() # ④ backward optimizer.step() # ⑤ update
3 — Batch Size, Epochs & Iterations
EpochOne full pass through the entire training set
Batch sizeSamples per gradient update. Smaller = noisier gradients, better generalisation. Larger = stable, needs more memory.
IterationOne gradient update. Iterations/epoch = dataset_size / batch_size
Mini-batch SGDBatch size 32–512 — standard. Combines efficiency of batch GD with noise benefits of SGD.
Start with: batch_size=32, train until validation loss stops improving (early stopping with patience=10).
4 — Regularization Techniques

Prevent overfitting (model memorises training data, fails on new data):

  • L2 (Weight Decay): Penalises large weights → optimizer = Adam(lr, weight_decay=1e-4)
  • L1: Encourages sparse weights (some exactly zero)
  • Dropout: Randomly zero out neurons during training. Prevents co-adaptation. Rate 0.2–0.5.
  • Batch Normalisation: Normalises activations per batch. Stabilises training, reduces sensitivity to learning rate. Use after linear layer, before activation.
  • Early Stopping: Stop when validation loss stops improving. Most effective regulariser.
  • Data Augmentation: Artificially expand training set (flip, crop, noise).

Optimizers & The Learning Objective

Before choosing an optimizer, understand what it's actually doing — and why that problem is harder than it looks.

The Core Optimization Objective

Every machine learning model — whether linear regression, a random forest, or a billion-parameter transformer — is ultimately solving the same mathematical problem:

minθ  L(θ)
SymbolMeaning
θ (theta)All trainable parameters — every weight and bias in the model
L(θ)Loss function — a scalar measuring how wrong the model's predictions are given the current θ
minFind the θ that makes L as small as possible

For a neural network with millions of parameters, θ is a vector in a millions-dimensional space. The loss surface is a landscape over that space — training is the process of navigating downhill to the lowest point you can find.

Common loss functions (what L(θ) actually is):

TaskLossFormula
RegressionMSE½ · mean((y − ŷ)²)
Regression (robust)HuberMSE for small errors, MAE for large
Binary classificationBinary Cross-Entropy−[y·log(ŷ) + (1−y)·log(1−ŷ)]
Multi-classCross-Entropy−Σ yᵢ · log(ŷᵢ)
RankingHinge Lossmax(0, margin − score)
Generative (VAE)ELBOReconstruction − KL divergence
The loss is not the metric. You minimise the loss (a smooth, differentiable proxy) and evaluate the metric (accuracy, F1, BLEU). Cross-entropy loss minimised → accuracy maximised, but they are not the same number. Always report the metric to stakeholders, never the raw loss.
Gradient Descent — How We Minimise L(θ)

We can't search every possible θ directly — the space is astronomically large. Instead we use the gradient: the direction in θ-space that increases L the most. Move opposite to that direction (downhill) by a small step.

θ ← θ − η · ∇θ L(θ)
SymbolMeaning
η (eta)Learning rate — step size. Too big: overshoots. Too small: slow.
∇θ LGradient of loss w.r.t. parameters — computed by backpropagation
← (update)Applied once per batch (not once per epoch)

This single update rule, repeated thousands to millions of times over mini-batches of data, is how every neural network learns.

Three flavours of gradient descent:

VariantBatch SizeGradient QualitySpeed
Batch GDFull datasetExact — no noiseVery slow; impractical at scale
Stochastic GD (SGD)1 sampleVery noisyFast per step; noisy updates can escape local minima

# PyTorch gradient descent — one update step optimizer.zero_grad() # clear old gradients loss = criterion(model(xb), yb) # forward: compute L(θ) loss.backward() # backward: compute ∇θ L optimizer.step() # update: θ ← θ − η∇θL
zero_grad() is critical. PyTorch accumulates gradients by default. Forgetting to call zero_grad() is one of the most common training bugs — gradients from previous batches pile up and the update is wrong.
Local Minima, Global Minima & Saddle Points

The loss surface of a neural network is not a bowl (convex) — it is a highly complex landscape with many valleys, ridges, and flat regions. Gradient descent can get stuck.

FeatureWhat It IsProblem?
Global minimumLowest possible L(θ) — best solutionRarely found exactly in practice
Local minimumValley — lower than all neighbours, but not globally lowestCan trap gradient descent
Saddle pointFlat region — gradient ≈ 0 but not a minimumMore common than local minima in high dimensions; slows training
PlateauBroad flat area of similar lossLearning appears to stop; often needs LR warmup or schedule change

The surprising truth about local minima in deep learning:

In very high-dimensional parameter spaces (millions of θ), true local minima are extremely rare. A point that is a minimum in every direction simultaneously is vanishingly unlikely. Most apparent local minima are actually saddle points. Research (Goodfellow et al., 2015) showed that local minima in deep nets tend to have similar loss values to the global minimum — so escaping them is less critical than it first appears.

Practical techniques to escape / avoid bad minima:

  • Momentum — accumulates velocity across steps, rolls through small valleys
  • Stochastic noise — mini-batch gradient noise acts as natural perturbation
  • Learning rate warmup — start slow, then accelerate past initial bad regions
  • Cyclical / cosine LR schedules — periodic LR increases help escape plateaus
  • Random restarts — train from multiple random initialisations, take best
  • Good initialisation — Kaiming/He init places starting θ in a better basin
Flat minima generalise better. Sharp minima (narrow valleys) tend to overfit — a tiny shift in test data shoots loss upward. Flat minima are robust to small perturbations. SGD with momentum tends to find flatter minima than Adam, which may explain why SGD sometimes generalises better despite converging slower.

Learning Rate Schedulers

Adjust learning rate during training for faster convergence and better final accuracy.

  • Step Decay: Multiply LR by γ every N epochs
  • Cosine Annealing: Smoothly decays LR following cosine curve — standard for transformers
  • Warmup: Ramp LR from 0 to target over first N steps — critical for large models
  • ReduceLROnPlateau: Reduce when val loss stops improving
  • One-Cycle Policy: LR rises then falls — fast training, often best result

Hyperparameter Tuning

Finding the best configuration for your model — learning rate, batch size, depth, regularization — without manually guessing.

Grid Search Beginner

Exhaustively evaluates every combination of the hyperparameter grid. Simple and reproducible, but cost grows exponentially with the number of parameters.

from sklearn.model_selection import GridSearchCV from sklearn.ensemble import RandomForestClassifier param_grid = { 'n_estimators': [100, 300], 'max_depth': [None, 10, 20], 'min_samples_split': [2, 5] } gs = GridSearchCV(RandomForestClassifier(), param_grid, cv=5, scoring='f1_macro', n_jobs=-1) gs.fit(X_train, y_train) print(gs.best_params_)

When to use: Small grids (<50 combos), final polish after narrowing range.

Random Search Beginner

Samples random combinations from continuous distributions. Finds good solutions 2–10× faster than grid search because most hyperparameters are not equally important.

from sklearn.model_selection import RandomizedSearchCV from scipy.stats import loguniform, randint param_dist = { 'learning_rate': loguniform(1e-4, 1e-1), 'n_estimators': randint(50, 500), 'max_depth': randint(3, 15) } rs = RandomizedSearchCV(model, param_dist, n_iter=60, cv=5, n_jobs=-1) rs.fit(X_train, y_train)

Rule of thumb: 60 random trials achieves ~95% coverage of a well-sampled grid.

Bayesian Optimization Intermediate

Builds a probabilistic surrogate model (Gaussian Process or Tree Parzen Estimator) of the objective surface, then samples where improvement is most likely.

  • Surrogate is updated after each trial
  • Acquisition function (EI, UCB) picks next point
  • Converges in 20–50 trials vs 100s for random
  • Best for expensive evaluations (GPU hours)

Libraries: scikit-optimize, hyperopt, Optuna

Optuna Intermediate

Modern standard for HPO. Uses TPE with pruning — cancels poor trials early to save compute. Dashboard built in.

import optuna def objective(trial): lr = trial.suggest_float('lr', 1e-5, 1e-2, log=True) wd = trial.suggest_float('wd', 1e-6, 1e-3, log=True) lay = trial.suggest_int('layers', 2, 6) return train_model(lr, wd, lay) # returns val_loss study = optuna.create_study(direction='minimize') study.optimize(objective, n_trials=100) print(study.best_params)

Cross-Validation Strategies

k-Fold CV (k=5 or 10) — standard for tabular data. Rotate which fold is the validation set, average the k scores. More reliable than a single train/val split.

from sklearn.model_selection import StratifiedKFold, cross_val_score skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42) scores = cross_val_score(model, X, y, cv=skf, scoring='f1_macro') print(f'CV F1: {scores.mean():.3f} ± {scores.std():.3f}')
  • Stratified k-Fold — preserves class ratio in each fold (use for classification)
  • GroupKFold — prevents data from the same subject leaking across folds
  • TimeSeriesSplit — walk-forward validation for temporal data (never shuffle)
  • Nested CV — outer loop for performance estimate, inner for HPO; avoids optimistic bias
Adversarial & Edge Case Testing

Probe the model with inputs designed to break it — not to attack it, but to find blind spots before deployment does.

  • FGSM / PGD attacks — add imperceptible pixel noise that flips image classifier output
  • TextFooler / BERT-Attack — substitute synonyms to fool NLP models
  • Boundary samples — inputs near the decision boundary expose fragility
  • Out-of-distribution (OOD) — test on data from a different domain, season, or source
  • Null inputs — all-zeros, all-ones, empty strings, NaN-filled rows

Target: Graceful degradation, not crashes — model should return a low-confidence prediction rather than an exception.

Bias & Fairness Auditing

A model accurate on average may be systematically wrong for a subgroup. Always slice your test metrics by sensitive attributes.

  • Demographic parity — positive rate equal across groups
  • Equalized odds — equal TPR and FPR across groups
  • Calibration by subgroup — predicted probability matches actual frequency
# Slice test set by subgroup for grp, sub in test_df.groupby('gender'): preds = model.predict(sub[features]) f1 = f1_score(sub['label'], preds) print(f'{grp}: F1={f1:.3f}')

Tools: Fairlearn, IBM AI Fairness 360, What-If Tool

Latency & Throughput Profiling

A model that takes 800ms per prediction is unusable in a real-time product. Profile before you deploy.

import time, torch model.eval() # Warm up GPU with torch.no_grad(): for _ in range(10): model(dummy_input) # Measure 100 runs times = [] with torch.no_grad(): for _ in range(100): t0 = time.perf_counter() model(dummy_input) times.append(time.perf_counter() - t0) print(f'p50={sorted(times)[50]*1000:.1f}ms') print(f'p99={sorted(times)[99]*1000:.1f}ms')

Pre-Deploy Testing Checklist
  • Test set held out until final evaluation — never used during training or tuning
  • Cross-validation score and held-out score agree (±3% is normal; >5% gap = overfit)
  • Calibration curve plotted — predicted probabilities match real frequencies
  • OOD test set evaluated — how does accuracy degrade on unseen distributions?
  • Performance sliced by all known sensitive attributes
  • p50 and p99 latency measured under realistic batch sizes
  • Memory footprint profiled — model fits on target hardware
  • Null / malformed inputs return graceful output (no exceptions)
  • Model versioned and artifacts saved (weights + config + preprocessing)
  • Monitoring set up for data drift and prediction distribution shift

Common Pitfalls

These mistakes are responsible for most failed ML projects. Knowing them in advance is half the solution.

The central challenge of machine learning is finding the Goldilocks zone of model complexity:

  • A too-simple model (linear regression on a non-linear problem) has high bias — it makes systematic errors because it can't represent the true pattern.
  • A too-complex model (a 50-layer network on 500 samples) has high variance — it fits the training noise perfectly but fails on new data.
  • A just-right model learns the underlying signal and ignores noise — its training and validation metrics are close to each other.

This is formally called the Bias-Variance Tradeoff. Every modelling decision — architecture depth, regularisation, dataset size — moves you along this spectrum. The tools to find the sweet spot: cross-validation, learning curves, and regularisation.

Overfitting

Symptom: Training accuracy 98%, validation accuracy 72%. The model memorised the training set.

Fixes (in order of preference):

  1. Get more training data (most effective)
  2. Add Dropout (0.2–0.5 after each hidden layer)
  3. L2 regularisation (weight decay)
  4. Reduce model capacity (fewer layers / smaller hidden size)
  5. Data augmentation (images: flip, crop, colour jitter)
  6. Early stopping on validation loss
Never evaluate augmented data in your validation set — augmentation is training-only.
Underfitting

Symptom: Both training and validation accuracy are poor. The model has too little capacity or too little training.

Fixes:

  1. Increase model capacity (more layers / wider hidden sizes)
  2. Train longer (more epochs, check loss is still declining)
  3. Reduce regularisation (lower dropout, lower weight decay)
  4. Use a more expressive architecture (e.g., deep net vs. linear)
  5. Engineer better features — linear models underfit complex relationships
  6. Lower learning rate if loss is oscillating
Data Leakage

Future information leaks into training features, causing unrealistically high scores that collapse in production.

Common sources:

  • Scaling / normalisation fit on the full dataset before splitting
  • Target encoding computed on the whole dataset
  • Time-series: using future timestamps as features
  • Group leakage: same patient/user in train and test
Rule: Fit all preprocessing transformers on training data only. Apply (transform-only) to validation and test.
Vanishing & Exploding Gradients

In deep networks, gradients can shrink to zero (vanish) or grow uncontrollably (explode) during backprop, stalling training.

Vanishing fixes:

  • ReLU / Leaky ReLU / GELU instead of sigmoid/tanh
  • Residual connections (ResNet skip connections)
  • Batch Normalisation after linear layers
  • Careful weight initialisation (Kaiming / He init for ReLU)

Exploding fixes:

  • Gradient clipping: torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
  • Reduce learning rate
Class Imbalance

When 99% of examples are class 0, a model that predicts class 0 every time scores 99% accuracy — but is useless.

Fixes (ranked by effectiveness):

  1. Class weights in loss: CrossEntropyLoss(weight=class_weights)
  2. Oversampling minority: SMOTE (for tabular), duplicate with augmentation (images)
  3. Undersampling majority: Random or Tomek Links
  4. Threshold tuning: Default 0.5 threshold is wrong — tune on PR curve
  5. Use PR-AUC not accuracy as your metric

A complete end-to-end workflow from raw idea to deployed model. Follow this order — skipping steps causes pain later.

Define the Problem Precisely

ML is an optimisation tool — it needs a clear objective. Answer these before writing any code:

  • What exactly are you predicting? (label, value, cluster, sequence)
  • What does success look like? (metric + threshold: "AUC > 0.85 on held-out set")
  • What data exists, and what is the ground truth label?
  • What are the constraints? (latency, model size, interpretability, cost)
  • Is ML actually necessary? Can a rule-based system solve 80% of cases?

  • Calibration curve is reasonable

Feature Engineering

The single highest-leverage skill in applied ML. A mediocre model on great features beats a great model on raw data every time. "Garbage in, garbage out" is not a cliché — it's the root cause of most failed projects.

Numeric Features

Scaling — why it matters:

Algorithms that use distance (KNN, SVM) or gradient descent (neural nets, logistic regression) are sensitive to feature scale. A feature ranging 0–1,000,000 will dominate one ranging 0–1.

Feature Creation & Interactions

Creating new features from existing ones is often more valuable than tuning the model. Domain knowledge drives this.

  • Ratio features: price / sqft, clicks / impressions (CTR)
  • Interaction terms: age × income, width × height
  • Aggregations: user's 7-day mean spend, rolling std
  • Date decomposition: hour, day_of_week, is_weekend, days_since
  • Binning: age → [0–18, 18–35, 35–65, 65+] buckets
  • Clustering as feature: assign each row its k-means cluster ID

Data Pipelines

How data moves from raw source to model input — and how to make that process fast, reproducible, and leakage-free.

sklearn Pipeline — The Right Way

A Pipeline chains preprocessing steps and the model into one object. It guarantees no leakage (fit only on training data), enables cross-validation on the full pipeline, and makes deployment trivial — save one object, serve one object.

from sklearn.pipeline import Pipeline from sklearn.compose import ColumnTransformer from sklearn.preprocessing import StandardScaler, OneHotEncoder from sklearn.impute import SimpleImputer from xgboost import XGBClassifier num_pipe = Pipeline([ ('impute', SimpleImputer(strategy='median')), ('scale', StandardScaler()), ]) cat_pipe = Pipeline([ ('impute', SimpleImputer(strategy='most_frequent')), ('encode', OneHotEncoder(handle_unknown='ignore')), ]) pre = ColumnTransformer([ ('num', num_pipe, num_cols), ('cat', cat_pipe, cat_cols), ]) model = Pipeline([ ('prep', pre), ('clf', XGBClassifier()), ]) model.fit(X_train, y_train) # fit once model.predict(X_test) # use everywhere
PyTorch Dataset & DataLoader

For deep learning, PyTorch separates data loading (Dataset) from batching and shuffling (DataLoader). This enables parallel loading with multiple CPU workers while the GPU trains.

Train / Val / Test Split Discipline
  • Train (70–80%) — model sees this during training
  • Validation (10–15%) — used for HPO and early stopping. Model never trains on it but you indirectly use it.
  • Test (10–15%) — touched exactly once, at the very end. Reports real-world performance.
If you tune hyperparameters on the test set, your test score is optimistic. The test set must be invisible until final evaluation.

For time series: always split by time — never shuffle. Future data cannot inform past predictions.

Data Augmentation

Artificially expand the training set by applying label-preserving transformations. Reduces overfitting. Never augment validation or test data.

Images:

  • Flip, crop, rotate, colour jitter, cutout, mixup
  • torchvision.transforms or albumentations

Text (NLP):

  • Back-translation, synonym swap, random deletion
  • nlpaug library

Tabular:

  • SMOTE (oversample minority class), Gaussian noise injection

Transfer Learning & Fine-Tuning

Training a model from scratch requires millions of labelled examples and days of GPU time. Transfer learning lets you start from a powerful pretrained model and adapt it to your task in hours — sometimes with only a few hundred examples.

The Core Idea

A model pretrained on a large dataset (ImageNet, Common Crawl, GitHub code) has already learned general representations — edges, textures, grammar, semantics. Your task just needs the final layers retrained to your specific labels.

ScenarioStrategy
Small dataset, similar domainFreeze all layers, retrain head only
Medium dataset, similar domainFine-tune last 2–4 layers + head
Large dataset, different domainFine-tune full model with low LR
Very different domainUse pretrained as init, train all
import torchvision.models as models # Load pretrained ResNet50 model = models.resnet50(weights='IMAGENET1K_V2') # Freeze all layers for param in model.parameters(): param.requires_grad = False # Replace final layer for your task (e.g. 5 classes) model.fc = nn.Sequential( nn.Dropout(0.3), nn.Linear(model.fc.in_features, 5) ) # Only model.fc parameters will be updated optimizer = torch.optim.AdamW( model.fc.parameters(), lr=1e-3)
Use a lower LR for pretrained layers than the new head — e.g., head LR = 1e-3, backbone LR = 1e-5. This prevents "catastrophic forgetting" of pretrained knowledge.

Explainability & Interpretability (XAI)

A model you can't explain is a model you can't trust in production — especially in regulated industries (finance, healthcare, legal). XAI bridges the gap between model accuracy and human understanding.

Interpretable vs Explainable — The Distinction

Intrinsically interpretable models:

Transparent by design — you can read the decision directly from the model itself.

  • Linear / Logistic Regression — each coefficient = effect of that feature on output
  • Decision Tree — follow the branch path to the leaf
  • Rule-based models — explicit IF-THEN logic
  • GAMs (Generalised Additive Models) — f(x) = f₁(x₁) + f₂(x₂) + … smooth, interpretable

Post-hoc explainability (black-box models):

Applied after training to explain individual predictions or global model behaviour. Works on any model.

  • SHAP — game-theoretic, model-agnostic, theoretically grounded
  • LIME — local linear approximation around one prediction
  • Permutation importance — shuffle a feature, measure accuracy drop
  • Attention maps — visualise which tokens/pixels the model focused on
  • Grad-CAM — gradient-weighted class activation maps for CNNs
SHAP — SHapley Additive exPlanations Intermediate

SHAP assigns each feature a Shapley value — its fair contribution to the prediction, averaging over all possible feature orderings. The only method that satisfies all four fairness axioms (efficiency, symmetry, dummy, additivity).

Time Series Forecasting

Time series data is ordered — past observations influence future ones. Standard ML assumptions (i.i.d. samples) break down. You need specialised models and validation strategies.

Never shuffle time series data. Splitting randomly creates data leakage — future values used to predict the past. Always split by time: train on earlier data, validate on later data.

Linear Algebra — The Language of ML

Every operation in a neural network is a matrix multiplication. Every embedding is a vector. Every gradient is a Jacobian. Linear algebra is not background knowledge — it is machine learning.

Eigenvalues & Eigenvectors

For matrix A: Av = λv — eigenvector v is unchanged in direction when multiplied by A; λ is the scaling factor.

Why it matters in ML:

  • PCA: eigenvectors of the covariance matrix are the principal components; eigenvalues = explained variance
  • Spectral clustering: eigenvectors of graph Laplacian define cluster structure
  • Hessian: eigenvalues of the loss Hessian reveal curvature — large eigenvalues = sharp minima = overfitting risk
  • Gradient stability: weight matrices with eigenvalues >1 cause exploding gradients; <1 cause vanishing
import numpy as np A = np.array([[4,2],[1,3]]) vals, vecs = np.linalg.eig(A) # vals: eigenvalues λ # vecs: columns are eigenvectors
SVD — Singular Value Decomposition

A = UΣVᵀ — any matrix decomposes into rotation U, scaling Σ (diagonal), rotation Vᵀ. The diagonal values of Σ are singular values (always ≥ 0).

ML applications:

  • PCA: SVD of centred data X gives principal components — no need to form XᵀX explicitly
  • Recommendation (MF): decompose user-item matrix into latent factors
  • Low-rank approximation: keep top-k singular values → compressed representation (LoRA is exactly this)
  • Pseudo-inverse: A⁺ = VΣ⁺Uᵀ — solves least squares when A is not invertible
U, s, Vt = np.linalg.svd(A, full_matrices=False) # Low-rank approx with top k components k = 10 A_approx = U[:,:k] @ np.diag(s[:k]) @ Vt[:k,:]
PCA — Math from First Principles

PCA finds the directions of maximum variance in data. Step by step:

  1. Centre data: X_c = X − μ
  2. Compute covariance: C = (1/n) X_cᵀX_c
  3. Eigen-decompose: C = QΛQᵀ
  4. Sort eigenvectors by eigenvalue (descending)
  5. Project: Z = X_c · Q[:, :k]

Explained variance ratio: λᵢ / Σλⱼ — tells you what % of total variance each component captures.

from sklearn.decomposition import PCA pca = PCA(n_components=2) Z = pca.fit_transform(X_scaled) print(pca.explained_variance_ratio_) # [0.72, 0.18] → 90% variance in 2 dims

A single dense layer, written out fully:

Hypothesis Testing

Used to decide if observed model improvements are real or due to random chance.

Workflow:

  1. State H₀ (null) and H₁ (alternative)
  2. Choose significance level α (usually 0.05)
  3. Compute test statistic and p-value
  4. If p < α: reject H₀ (result is significant)

Common tests in ML:

  • t-test: compare means of two models' CV scores
  • McNemar's test: compare two classifiers on same test set
  • Wilcoxon signed-rank: non-parametric alternative to t-test
  • Bootstrap test: permutation test for any metric
p-value ≠ effect size. A model that's 0.001% better can have p<0.05 on a large test set. Always report confidence intervals alongside p-values.
Central Limit Theorem & Confidence Intervals

CLT: The sample mean of n i.i.d. samples converges to a normal distribution as n→∞, regardless of the underlying distribution: x̄ ~ N(μ, σ²/n)

Why it matters in ML:

  • Justifies using z-tests/t-tests on model performance metrics
  • Mini-batch gradient estimates are approximately normal at large batch sizes
  • Cross-validation scores are asymptotically normal

95% Confidence Interval for accuracy:

# Wilson score interval (better for proportions) from statsmodels.stats.proportion import proportion_confint ci_lo, ci_hi = proportion_confint( correct, total, alpha=0.05, method='wilson') print(f'Accuracy: {correct/total:.3f} [{ci_lo:.3f}, {ci_hi:.3f}]')
Correlation vs Causation

ML models learn correlations in data. They do not discover causal relationships. This distinction is critical for deployment.

Pearson correlation:

r = Σ(xᵢ-x̄)(yᵢ-ȳ) / (n·σₓσᵧ) — measures linear relationship. Range: [−1, 1]

Spearman rank correlation:

ρ = 1 − 6Σdᵢ²/n(n²−1) — measures monotonic relationship. Robust to outliers.

Confounders — why correlation misleads:

  • Ice cream sales correlate with drowning → confounder: hot weather
  • Hospital visits correlate with death → selection bias
  • Simpson's Paradox: trend reverses when data is disaggregated by group

Statistical Learning Theory

The formal mathematical framework answering: When does a model generalise? How much data do we need? What is the fundamental limit of learning?

The Generalisation Problem

We train on a finite sample S drawn from an unknown distribution D. We want low error on D, but can only measure error on S. The generalisation gap is:

generalisation gap = R(h) − R̂(h)

Where R(h) = true risk (error on D) and R̂(h) = empirical risk (error on S).

PAC Learning — Probably Approximately Correct:

A learning algorithm A PAC-learns concept class C if, for any ε > 0 and δ > 0, given enough samples m, A outputs h with P[R(h) ≤ ε] ≥ 1 − δ.

In plain English: with enough data, we can find a hypothesis that is approximately correct with high probability.

The required sample size m grows with model complexity and shrinks with ε and δ. This is why complex models need more data.
VC Dimension & Model Complexity

The VC dimension (Vapnik-Chervonenkis) of a hypothesis class H is the largest set of points it can shatter (classify in all possible ways).

ModelVC Dim
Threshold classifier on ℝ1
Linear classifier in ℝᵈd + 1
Decision tree (depth k)2ᵏ
Neural networkO(W log W) — W = weights
SVM (RBF kernel)∞ (always shatters)

Fundamental theorem of learning:

With probability ≥ 1−δ: R(h) ≤ R̂(h) + √(VC_dim·log(m/VC_dim) + log(1/δ)) / m. Higher VC dimension → larger generalisation bound → need more data to guarantee generalisation.

Bias-Variance Decomposition (Formal)

Expected prediction error decomposes into three terms:

E[(y − ŷ)²] = Bias² + Variance + Noise
  • Bias² = (E[ŷ] − y)² — systematic error from wrong assumptions
  • Variance = E[(ŷ − E[ŷ])²] — sensitivity to training set fluctuations
  • Noise = irreducible error from randomness in y

Complex models: low bias, high variance. Simple models: high bias, low variance. Regularisation trades variance for bias. Ensemble methods (bagging) reduce variance without increasing bias much.

No Free Lunch Theorem

The No Free Lunch (NFL) theorem (Wolpert, 1996): averaged over all possible problems, no learning algorithm outperforms any other.

What this really means:

  • There is no universally best ML algorithm
  • An algorithm that works on one class of problems performs poorly on another
  • Good performance requires inductive bias matching the problem structure
  • CNNs encode translation invariance — only good for spatial data
  • RNNs encode sequential order — only good for sequences
The NFL theorem is why we have many different model families — each encodes different inductive biases suited to different problem structures. Algorithm selection is domain expertise, not guessing.

The Transformer — How Attention Works (Math)

Scaled Dot-Product Attention:

Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V
  • Q (Query): "What am I looking for?"
  • K (Key): "What do I contain?"
  • V (Value): "What information do I carry?"
  • √d_k scaling: prevents dot products from growing too large → softmax becoming too peaked → vanishing gradients

Multi-Head Attention:

Run h attention heads in parallel, each with different learned projections W_Q, W_K, W_V. Concatenate outputs, project to model dimension. Different heads capture different relationship types (syntax, coreference, semantics).

import torch, torch.nn as nn, math class MultiHeadAttention(nn.Module): def __init__(self, d_model, n_heads): super().__init__() self.h = n_heads self.d_k = d_model // n_heads self.W_q = nn.Linear(d_model, d_model) self.W_k = nn.Linear(d_model, d_model) self.W_v = nn.Linear(d_model, d_model) self.W_o = nn.Linear(d_model, d_model) def forward(self, x, mask=None): B, T, D = x.shape # Project, reshape to (B, heads, T, d_k) Q = self.W_q(x).view(B,T,self.h,self.d_k).transpose(1,2) K = self.W_k(x).view(B,T,self.h,self.d_k).transpose(1,2) V = self.W_v(x).view(B,T,self.h,self.d_k).transpose(1,2) # Scaled dot-product attention scores = Q @ K.transpose(-2,-1) / math.sqrt(self.d_k) if mask is not None: scores = scores.masked_fill(mask, -1e9) attn = scores.softmax(dim=-1) out = (attn @ V).transpose(1,2).reshape(B,T,D) return self.W_o(out)
BERT — Encoder-Only Transformer

BERT (Bidirectional Encoder Representations from Transformers) reads the full sequence in both directions simultaneously.

Pretraining objectives:

  • Masked LM (MLM): 15% of tokens replaced with [MASK]. Model predicts original token. Forces bidirectional context.
  • Next Sentence Prediction (NSP): Classify if sentence B follows sentence A. (Later shown to be less useful.)

Fine-tuning tasks:

  • Classification: add linear layer on [CLS] token
  • NER/Tagging: add linear layer on each token
  • QA: predict start/end span positions
GPT — Decoder-Only Transformer

GPT (Generative Pretrained Transformer) uses causal (left-to-right) attention — each token can only attend to past tokens.

Pretraining objective:

  • Causal LM: predict next token given all previous. P(token_t | token_1…token_{t-1}). Trained on internet-scale text.

Key design choices:

  • Causal mask: upper triangle of attention matrix set to −∞
  • No encoder — decoder only, no cross-attention
  • Emergent in-context learning at scale (few-shot)

BERT vs GPT:

BERT = read understanding (classification, NER, QA). GPT = generation (chat, code, summarisation, translation).

Positional Encoding & Transformer Stack

Attention has no notion of order — "the cat sat" and "sat the cat" produce identical attention scores. Positional encoding injects position information.

Sinusoidal (original Transformer):

PE[pos, 2i] = sin(pos / 10000^(2i/d)) PE[pos, 2i+1] = cos(pos / 10000^(2i/d)) # Different frequencies → unique pattern per position # Allows model to attend by relative position

Modern alternatives:

  • Learned absolute: GPT-2, BERT — trainable embedding per position
  • RoPE (Rotary): LLaMA, GPT-NeoX — encodes relative positions via rotation; generalises to longer sequences
  • ALiBi: adds linear bias to attention scores; simple, extrapolates well

Generative Models — VAE, GAN, Diffusion

Generative models learn the data distribution p(x) and can sample new data from it. Understanding the math behind each family reveals why they produce different kinds of outputs — and why diffusion models produce the sharpest images.

VAE — Variational Autoencoder

VAE learns a probabilistic latent space — every input maps to a distribution, not a single point. This enables smooth interpolation and generation.

The ELBO objective:

L = E[log p(x|z)] − D_KL(q(z|x) ‖ p(z))
  • E[log p(x|z)] — reconstruction term: how well can we recover x from z?
  • D_KL(q ‖ p) — regularisation: keep latent posterior close to Gaussian prior N(0,I)

Reparameterisation trick:

# Sample z = μ + σ·ε, ε ~ N(0,I) # Allows gradients to flow through sampling eps = torch.randn_like(sigma) z = mu + sigma * eps
GAN — Generative Adversarial Network

Two networks compete in a minimax game: the Generator G tries to fool the Discriminator D; D tries to distinguish real from fake.

min_G max_D E[log D(x)] + E[log(1−D(G(z)))]

Training challenges:

  • Mode collapse: G generates only a few modes of p(x)
  • Training instability: D and G must stay in balance
  • Vanishing gradients: when D is too good, G gets no gradient

Fixes — modern GAN improvements:

  • Wasserstein GAN (WGAN): use Earth Mover distance, clip weights or gradient penalty
  • StyleGAN2: progressive growing, style injection, path length regularisation
  • Spectral normalisation on D layers
Diffusion Models

Diffusion models learn to reverse a gradual noising process. They currently produce the best image quality (DALL-E 3, Stable Diffusion, Midjourney).

Forward process (fixed, no learning):

q(xₜ|xₜ₋₁) = N(xₜ; √(1−βₜ)xₜ₋₁, βₜI)

Add Gaussian noise at each of T steps. At t=T, xₜ ≈ N(0,I).

Reverse process (learned):

pθ(xₜ₋₁|xₜ) = N(xₜ₋₁; μθ(xₜ,t), Σθ(xₜ,t))

A U-Net predicts the noise ε added at each step. Training loss: ‖ε − εθ(xₜ, t)‖²

Key components:

  • DDPM: original formulation, ~1000 steps
  • DDIM: deterministic, 50 steps (20× faster)
  • Classifier-free guidance: condition on text prompt during training by randomly dropping text; scale guidance at inference for quality/diversity tradeoff
Which generative model when: VAE → smooth latent space, useful for anomaly detection and representation learning. GAN → high-fidelity images, fast sampling (1 forward pass). Diffusion → best quality, slow (50–1000 steps), dominant for image/audio/video generation. Autoregressive (GPT) → best for text and discrete data.

Ensemble Methods — Theory & Practice

Ensembles combine multiple weak learners to build a strong learner. The mathematical intuition: if each model makes independent errors, averaging their predictions cancels much of the noise.

Bagging — Variance Reduction

Bootstrap Aggregating. Train B models on B different bootstrap samples (drawn with replacement). Average predictions.

ŷ = (1/B) Σᵦ hᵦ(x)

Why it works — mathematically:

Variance of average of B uncorrelated models = σ²/B. Even partially correlated models get significant variance reduction.

  • Doesn't reduce bias — base model must be low-bias
  • Each bootstrap sample contains ~63.2% unique examples (Poisson)
  • The ~36.8% left out = out-of-bag (OOB) estimate — free validation!
from sklearn.ensemble import BaggingClassifier model = BaggingClassifier( estimator=DecisionTreeClassifier(max_depth=None), n_estimators=100, oob_score=True) model.fit(X_train, y_train) print(model.oob_score_) # free OOB accuracy
Boosting — Bias Reduction

Train models sequentially, each focusing on the mistakes of the previous ones. Reduces bias by combining many weak learners into one strong one.

AdaBoost — adaptive weights:

  1. Initialise uniform sample weights wᵢ = 1/n
  2. Train weak learner hₜ on weighted data
  3. Compute error εₜ = Σwᵢ·1[hₜ(xᵢ)≠yᵢ]
  4. Compute model weight αₜ = ½ ln((1−εₜ)/εₜ)
  5. Update: wᵢ ← wᵢ·exp(−αₜyᵢhₜ(xᵢ)), normalise
  6. Final: H(x) = sign(Σ αₜhₜ(x))

Gradient Boosting — generalises AdaBoost:

Each tree fits the negative gradient of the loss — the residuals for MSE, pseudo-residuals for other losses.

# Gradient boosting step (pseudocode) # F₀(x) = initial prediction (mean for MSE) # For t = 1..T: # rᵢ = −∂L/∂F(xᵢ) ← negative gradient # hₜ = tree fit on {xᵢ, rᵢ} # Fₜ(x) = Fₜ₋₁(x) + η·hₜ(x)

Stacking (Stacked Generalisation):

Train diverse base models, then train a meta-learner on their out-of-fold predictions. The meta-learner learns which base models to trust on which inputs.

  • Soft voting: average predicted probabilities — almost always better

Gaussian Processes

A Gaussian Process (GP) is a distribution over functions. Instead of learning point estimates of parameters, a GP provides a full predictive distribution — mean + uncertainty — at every input point.

f(x) ~ GP(m(x), k(x,x'))

where m(x) = mean function, k(x,x') = kernel (covariance) function.

Common kernels:

  • RBF/Squared Exponential: k(x,x') = σ²exp(−‖x−x'‖²/2l²) — infinitely differentiable, very smooth
  • Matérn 5/2: twice differentiable — more realistic for real physical processes
  • Periodic: k encodes cyclic patterns — good for time series
from sklearn.gaussian_process import GaussianProcessRegressor from sklearn.gaussian_process.kernels import RBF, Matern gp = GaussianProcessRegressor( kernel=Matern(nu=2.5), n_restarts_optimizer=5) gp.fit(X_train, y_train) y_pred, y_std = gp.predict(X_test, return_std=True) # y_pred: mean prediction, y_std: ±1σ uncertainty

GP uses: Bayesian optimisation surrogate model (Optuna uses TPE, not GP, but GP-based BO is also common), small-data regression with uncertainty, spatial interpolation (Kriging).

Uncertainty in Deep Learning

Two types of uncertainty:

  • Aleatoric (data uncertainty): irreducible noise in the data itself — a blurry photo, an inherently ambiguous label. Cannot be reduced by collecting more data.
  • Epistemic (model uncertainty): uncertainty about which model/parameters are correct — reduces with more data. This is what Bayesian methods quantify.

Recommendation Systems

RecSys powers Netflix, Spotify, Amazon, TikTok, and YouTube. The core problem: given past interactions between users and items, predict what a user will engage with next.

Collaborative Filtering

Recommendations based purely on user-item interaction patterns — no item content needed. "Users like you also liked…"

User-based CF:

Find k most similar users (cosine / Pearson similarity on rating vectors). Predict rating as weighted average of their ratings.

Item-based CF:

Find similar items based on who rated them similarly. More stable than user-based (item preferences change slower than user tastes).

Problems with pure CF:

  • Cold start: new users/items have no interaction history
  • Sparsity: most users rate <1% of items
  • Popularity bias: popular items dominate recommendations
Matrix Factorisation

Decompose the user-item rating matrix R (m×n) into two low-rank matrices: R ≈ U·Vᵀ where U ∈ ℝᵐˣᵏ (user factors) and V ∈ ℝⁿˣᵏ (item factors), k ≪ min(m,n).

min_{U,V} Σ_{(i,j)∈Ω} (Rᵢⱼ − uᵢᵀvⱼ)² + λ(‖U‖²_F + ‖V‖²_F)
from surprise import SVD, Dataset, Reader reader = Reader(rating_scale=(1, 5)) data = Dataset.load_from_df(df[['user','item','rating']], reader) algo = SVD(n_factors=100, n_epochs=20, lr_all=0.005) # algo.predict(user_id, item_id) → estimated rating

ALS (Alternating Least Squares): Fix U, solve for V analytically. Fix V, solve for U. Repeat. Parallelisable, good for implicit feedback (clicks, plays) at scale — used by Spotify.

Ethics & Responsible AI

Building powerful models isn't enough — you must ensure they are fair, transparent, private, and accountable. Responsible AI is not a checklist; it's an engineering discipline.

Fairness Definitions

These definitions are often mutually exclusive — you cannot satisfy all simultaneously (Chouldechova 2017, COMPAS case). Choose based on the harm model.

Demographic Parity

Positive prediction rate is equal across groups:

P(Ŷ=1 | A=0) = P(Ŷ=1 | A=1)

Risk: may require predicting worse for high-risk individuals to balance rates. Good when base rates are equal.

Equalized Odds

Equal TPR and FPR across groups:

P(Ŷ=1|Y=y,A=0) = P(Ŷ=1|Y=y,A=1)  ∀y

Stronger than demographic parity. Equalized opportunity relaxes to TPR only. Common in hiring/lending.

Calibration

Predicted probability = actual frequency for all groups:

P(Y=1 | Ŷ=p, A=a) = p  ∀a, p

Critical for medical risk scores. Platt scaling / isotonic regression used post-hoc. COMPAS was calibrated but not equal odds.

Differential Privacy

ε-Differential Privacy (Dwork 2006)

A randomised mechanism M satisfies ε-DP if for all adjacent datasets D, D′ (differing in one record) and all outputs S:

P(M(D) ∈ S) ≤ e^ε · P(M(D′) ∈ S)
  • ε → 0: very strong privacy (more noise)
  • ε → ∞: no privacy guarantee
  • Typical practical range: ε ∈ [1, 10]
  • (ε,δ)-DP allows a δ probability of violation — used in practice
Mechanisms & DP-SGD

Gaussian Mechanism: Add noise σ = Δf·√(2ln(1.25/δ)) / ε where Δf is the L2 sensitivity of query f.

DP-SGD (Google): Clip per-sample gradients to norm C, add Gaussian noise, average — achieves (ε,δ)-DP training.

from opacus import PrivacyEngine privacy_engine = PrivacyEngine() model, optimizer, loader = privacy_engine.make_private_with_epsilon( module=model, optimizer=optimizer, data_loader=loader, epochs=10, target_epsilon=3.0, target_delta=1e-5, max_grad_norm=1.0)

Apple uses local DP (noise added on device). Google uses central DP (noise at aggregation). Both used in production at scale.

Federated Learning

FedAvg Algorithm

Train locally on each device, aggregate only model updates — raw data never leaves the device.

  1. Server sends global model weights θ to K clients
  2. Each client trains for E local epochs: θₖ ← θ − η·∇Lₖ(θ)
  3. Server aggregates: θ ← Σₖ (nₖ/n) · θₖ  (weighted by dataset size)
  4. Repeat for R rounds

Challenges: non-IID data (statistical heterogeneity), partial participation, communication cost, Byzantine clients. FedProx adds proximal term to handle heterogeneity.

GPT-style (decoder-only) processes tokens autoregressively — each token attends only to past tokens (causal mask).

Retrieval-Augmented Generation (RAG)

RAG Architecture

Augment LLM context with retrieved documents at inference time — no retraining needed, knowledge stays fresh.

  1. Indexing: Chunk documents → embed with bi-encoder → store in vector DB (Pinecone, Weaviate, pgvector, Chroma)
  2. Retrieval: Embed query → ANN search (HNSW/IVF) → top-k chunks by cosine similarity
  3. Augment: Inject retrieved chunks into prompt context
  4. Generate: LLM answers grounded in retrieved evidence

RLHF & Fine-Tuning

RLHF Pipeline
  1. SFT (Supervised Fine-Tuning): Fine-tune base LLM on curated instruction-response pairs using cross-entropy loss
  2. Reward Model: Train RM on human preference pairs (chosen vs rejected) using: L = −log σ(r(chosen) − r(rejected))
  3. PPO/GRPO: Optimise SFT model to maximise reward while staying close to SFT policy via KL penalty: r_total = r_RM − β·KL(π‖π_SFT)

DPO (Direct Preference Optimisation): Skips the RM entirely — optimises preference directly from comparison data. Simpler, often matches PPO quality.

# DPO loss (simplified) log_ratio_chosen = log_pi_chosen - log_pi_ref_chosen log_ratio_rejected = log_pi_rejected - log_pi_ref_rejected loss = -F.logsigmoid(beta * (log_ratio_chosen - log_ratio_rejected))

Pruning

Pruning Strategies

Remove weights/neurons/heads that contribute least to model output. Two main flavours:

  • Unstructured: Set individual weights to zero (magnitude pruning). Achieves 90%+ sparsity; needs sparse hardware/libraries to see speedup (sparse BLAS, NVIDIA Ampere structured sparsity).
  • Structured: Remove entire filters, heads, or layers. Immediately faster on dense hardware. Less aggressive compression.
import torch.nn.utils.prune as prune # Prune 30% of weights in a linear layer (L1 unstructured) prune.l1_unstructured(layer, name='weight', amount=0.3) # Make pruning permanent prune.remove(layer, 'weight')
Lottery Ticket Hypothesis

Frankle & Carlin (2019): Within a large randomly initialised network exists a small "winning ticket" subnetwork that, if trained from scratch at its original initialisation, can match the full network's accuracy.

Iterative Magnitude Pruning:

  1. Train full network to convergence
  2. Prune p% of smallest weights
  3. Reset remaining weights to their initial values
  4. Repeat from step 1

Practical impact: Winning tickets at 80–95% sparsity often match dense accuracy. Foundation for modern structured pruning research (SparseGPT, Wanda).

Knowledge Distillation

Response Distillation (Hinton 2015)

Train a small student model to mimic a large teacher model's output distribution, not just its hard labels.

L = α·L_CE(y_hard, ŷ_s) + (1−α)·T²·KL(σ(z_t/T) ‖ σ(z_s/T))

T (temperature): Higher T softens the teacher distribution, revealing inter-class relationships (e.g. "cat" and "dog" are more similar than "cat" and "car").

T = 4.0 soft_targets = F.softmax(teacher_logits / T, dim=-1) soft_preds = F.log_softmax(student_logits / T, dim=-1) distill_loss = F.kl_div(soft_preds, soft_targets, reduction='batchmean') * (T**2)

Causal Inference

ML models excel at prediction — but they can't distinguish cause from correlation. Causal inference lets you answer why and what if, not just what. Essential for A/B testing, policy decisions, and understanding model behaviour.

Causation vs Correlation

Why Correlation Misleads

Classic examples:

  • Ice cream sales ↑ drowning deaths ↑ — confounder: hot weather
  • Shoe size predicts reading ability in children — confounder: age
  • Hospital patients have worse outcomes — selection bias (sicker people go to hospitals)
  • More fire trucks → more damage — reverse causation

ML implication: A model trained on observational data learns correlations, not mechanisms. If you act on predictions (e.g., send discount to likely churners), the intervention changes the data distribution — your model breaks.

Potential Outcomes Framework (Rubin)

Each unit i has two potential outcomes: Y(1) (treated) and Y(0) (untreated). We only observe one.

ITE = Y_i(1) − Y_i(0)  (never both observable)
ATE = E[Y(1) − Y(0)] = E[Y(1)] − E[Y(0)]

Fundamental problem: We can only observe Y_i(T_i), not both counterfactuals. All causal methods try to estimate the missing counterfactual.

SUTVA assumption: No interference between units; treatment is well-defined.

Causal Graphs (DAGs)

Pearl's do-calculus uses Directed Acyclic Graphs to encode causal structure and identify confounders.

Key concepts:

  • Confounder: Variable causing both treatment and outcome (must control for)
  • Collider: Variable caused by both treatment and outcome (do NOT control — opens backdoor)
  • Mediator: Variable on the causal path (control blocks the effect you want to measure)
  • do-operator: P(Y|do(X=x)) ≠ P(Y|X=x) — intervention vs observation
  • Backdoor criterion: Block all non-causal paths from X to Y via conditioning

Causal Estimation Methods

Randomised Controlled Trials (RCTs)

The gold standard. Random assignment makes treatment independent of all confounders. A/B tests are RCTs.

ATE ≈ ȳ_treatment − ȳ_control

Power analysis: How many samples do you need?

from statsmodels.stats.power import TTestIndPower power_analysis = TTestIndPower() n = power_analysis.solve_power( effect_size=0.2, power=0.8, alpha=0.05) # n per group ≈ 394 for small effect (Cohen's d=0.2)

Limitations: Expensive, slow, often unethical. Online A/B tests suffer novelty effects, network effects (social platforms), and Hawthorne effects.

Difference-in-Differences (DiD)

Compare the change in outcomes before/after treatment between treated and control groups. Eliminates time-invariant confounders.

DiD = (Ȳ_T,post − Ȳ_T,pre) − (Ȳ_C,post − Ȳ_C,pre)

Parallel trends assumption: In the absence of treatment, treated and control groups would have followed the same trend. Test by checking pre-treatment trends.

import statsmodels.formula.api as smf model = smf.ols( 'outcome ~ treated + post + treated:post + controls', data=df).fit() did_estimate = model.params['treated:post']
Propensity Score Matching

Estimate P(T=1|X) — the probability of receiving treatment given covariates — then match treated and control units with similar scores.

  1. Fit logistic regression: P(T|X)
  2. Match each treated unit to closest control by PS
  3. Check covariate balance (standardised mean difference < 0.1)
  4. Estimate ATE on matched sample
from sklearn.linear_model import LogisticRegression ps_model = LogisticRegression().fit(X, T) propensity = ps_model.predict_proba(X)[:, 1] # Then match on propensity scores using causalml or MatchIt
Instrumental Variables (IV)

Use a variable Z (instrument) that affects treatment but affects outcome only through treatment — bypasses unobserved confounders.

LATE = Cov(Y,Z) / Cov(T,Z)

Classic examples:

  • Quarter of birth as IV for education (Angrist & Krueger)
  • Random encouragement as IV for actual treatment take-up
  • Distance to college as IV for college attendance

Causal Thinking in ML Practice

When Causal Matters
  • Churn models: Sending discounts to "likely churners" can train the model to identify who responds to discounts, not who would have churned
  • Recommendation: Did users enjoy this item because we showed it to them, or would they have found it anyway?
  • Credit scoring: Does approving a loan cause repayment, or do creditworthy people repay regardless?
  • Healthcare: Does a drug work, or are healthier patients more likely to be prescribed it?
  • Hiring: Do certain colleges produce better employees, or do better employees go to those colleges?
Causal ML Pitfalls
  • SHAP ≠ causal: Feature importance measures predictive contribution, not causal effect. Correlated features confuse it.
  • Overlap assumption: PSM fails if treated and control populations don't overlap in covariate space
  • Weak instruments: IV with F-stat < 10 is unreliable — amplifies bias
  • Parallel trends violation: DiD is invalid if treated group was trending differently before treatment
  • SUTVA violation: Network effects (social platforms) break standard causal estimates — use cluster or switchback experiments

Neural Architecture Search (NAS)

Automate the design of neural network architectures. Three main paradigms:

  • RL-based NAS (NASNet): Controller RNN generates architecture; reward = validation accuracy. Expensive — 2000 GPU-days originally.
  • Evolutionary NAS (AmoebaNet): Mutation + selection over architecture population. More parallelisable than RL.

  • One-shot / weight sharing: Train a supernet once; sub-networks inherit weights. (SNAS, SMASH, Once-for-All)
  • Zero-cost proxies: Score architectures at init (no training!) using gradient/activation statistics — 1000× faster than standard NAS.

Self-Supervised Learning

Self-supervised learning (SSL) creates supervisory signals from the data itself — no human labels required. It's the paradigm behind BERT, GPT, CLIP, MAE, and nearly every modern foundation model.

Why Self-Supervised?

The Labeling Bottleneck

Supervised learning needs expensive human labels. SSL exploits the structure already in data:

  • Predict the next word (GPT) — no labels needed
  • Reconstruct masked patches (MAE) — image is its own label
  • Match image to caption (CLIP) — web data is free
  • Predict rotation angle — transformation is label

Result: Models pre-trained on billions of unlabelled examples then fine-tuned on hundreds of labelled examples beat fully supervised models trained on millions.

Pretext vs Downstream Tasks

Pretext task: The self-supervised objective used during pre-training — a proxy designed to force the model to learn useful representations.

Downstream task: The actual task you care about — classification, detection, QA — solved by fine-tuning or linear probing on top of frozen SSL representations.

Linear probe: Freeze backbone, train only a linear classifier on top. Measures representation quality without confounding fine-tuning tricks.

Key insight: Good pretext tasks produce representations where downstream tasks are linearly separable — a strong signal that the model has learned semantic structure.

Contrastive Learning

SimCLR (Chen et al. 2020)

For each image, create two augmented views. Pull same-image pairs (positives) together, push all other pairs (negatives) apart in embedding space.

L = −log exp(sim(zᵢ,zⱼ)/τ) / Σₖ≠ᵢ exp(sim(zᵢ,zₖ)/τ)

This is NT-Xent (normalised temperature-scaled cross-entropy). τ (temperature) controls sharpness — typically 0.07–0.5.

Key ingredients:

  • Strong augmentations: random crop, colour jitter, Gaussian blur, grayscale
  • Projection head g(·): 2-layer MLP on top of backbone — improves linear probe by ~10%
  • Large batch (4096+): more negatives → better learning signal
# SimCLR NT-Xent loss (simplified) def nt_xent(z1, z2, tau=0.5): z = F.normalize(torch.cat([z1, z2]), dim=1) sim = z @ z.T / tau N = z1.shape[0] labels = torch.arange(N).roll(N).to(z.device) return F.cross_entropy(sim, labels)
MoCo & BYOL — No Large Batch Required

MoCo (He et al.): Maintains a queue of negatives from past batches + momentum encoder. Decouples batch size from number of negatives.

θ_k ← m·θ_k + (1−m)·θ_q  (momentum update, m=0.999)

BYOL (Bootstrap Your Own Latent): Eliminates negatives entirely — online network predicts EMA target network's representation. Requires stop-gradient on target.

L = 2 − 2 · (q(z_online) · z_target) / (‖q(z_online)‖ · ‖z_target‖)

Why BYOL doesn't collapse: The asymmetry (predictor + EMA target + stop-grad) prevents the trivial constant solution. Analysed via implicit regularisation and feature decorrelation.

DINO & Masked Autoencoders

DINO / DINOv2

Self-distillation with no labels. Student ViT processes local crops; teacher ViT (EMA of student) processes global crop. Student learns to match teacher's softmax output.

Why ViT + DINO works so well: Attention maps in DINO-trained ViTs spontaneously attend to semantic object boundaries — emergent segmentation without any supervision.

DINOv2 adds curated 142M image dataset + register tokens, achieving state-of-the-art on many dense prediction tasks (depth, segmentation) without fine-tuning.

Centering: Subtract running mean of teacher outputs to prevent collapse to one dominant dimension.

Masked Autoencoders (MAE)

He et al. 2021: Mask 75% of image patches, reconstruct pixel values of masked patches using a ViT encoder + lightweight decoder.

L = ‖pixels_masked − decoder(encoder(visible_patches))‖²

Key design choices:

  • 75% masking ratio (much higher than BERT's 15%) — harder task, better representations
  • Encoder only sees unmasked tokens — efficient (only 25% of compute)
  • Decoder is asymmetric and lightweight — discarded after pre-training
  • Predicts normalised pixel values per patch (not raw pixels)

NLP analogue: BERT uses 15% masking with WordPiece tokens. SpanBERT masks contiguous spans — harder, better for extraction tasks.

SSL in NLP: The Full Picture

MLM (BERT)
Mask 15% of tokens, predict originals. Bidirectional context. Best for classification, NER, extractive QA.
CLM (GPT)
Predict next token autoregressively. Causal attention mask. Best for generation. Scales to very long contexts.
Span Corruption (T5)
Replace contiguous token spans with sentinel tokens; decoder reconstructs them. Encoder-decoder. Best for seq2seq tasks.
RTD (ELECTRA)
Generator replaces tokens; discriminator predicts which tokens were replaced. 4× more efficient than MLM — trains on every token not just 15%.

Graph Neural Networks (GNNs)

Many real-world systems are graphs — molecules, social networks, knowledge bases, road networks, code ASTs. GNNs extend deep learning to this irregular structure via message passing between nodes.

Graph Fundamentals

Graph Representation

A graph G = (V, E) has nodes V and edges E. In ML, each node v has feature vector xᵥ and optionally each edge has features eᵤᵥ.

Adjacency matrix A: Aᵢⱼ = 1 if edge (i,j) exists. Sparse for real-world graphs.

Degree matrix D: Dᵢᵢ = Σⱼ Aᵢⱼ (number of neighbours).

Normalised Laplacian: L = I − D⁻½AD⁻½ — eigenvalues define graph frequency spectrum (used in spectral methods).

Types: Homogeneous, heterogeneous (multiple node/edge types), directed, weighted, temporal, hypergraphs.

Message Passing Framework (MPNN)

The general GNN recipe — each node aggregates messages from neighbours, then updates its own representation:

mᵥ⁽ˡ⁾ = AGG({MSG(hᵤ⁽ˡ⁻¹⁾, hᵥ⁽ˡ⁻¹⁾, eᵤᵥ) : u ∈ N(v)})
hᵥ⁽ˡ⁾ = UPDATE(hᵥ⁽ˡ⁻¹⁾, mᵥ⁽ˡ⁾)

After L layers, each node's representation captures its L-hop neighbourhood. Tasks:

  • Node-level: Classify each node (e.g., fraud detection)
  • Edge-level: Predict links (e.g., RecSys, knowledge graphs)
  • Graph-level: Classify whole graph (e.g., molecule property)
GNN Expressiveness

Weisfeiler-Leman (WL) test: Standard GNNs (GCN, GraphSAGE) are at most as expressive as the 1-WL graph isomorphism test — cannot distinguish certain non-isomorphic graphs.

GIN (Graph Isomorphism Network): Maximally powerful among standard MPNN — uses sum aggregation + MLP update:

hᵥ⁽ˡ⁾ = MLP((1+ε)·hᵥ⁽ˡ⁻¹⁾ + Σᵤ∈N(v) hᵤ⁽ˡ⁻¹⁾)

Beyond 1-WL: Higher-order GNNs (k-WL), random node features, structural encodings (distance, cycles) — needed for molecular property prediction.

GNN Challenges & Solutions

Over-smoothing

With many layers, all node representations converge to the same value — neighbourhood aggregation averages out features.

Solutions: Residual connections (JK-Net jumping knowledge), DropEdge (randomly drop edges per layer), pair-norm, initial residual connections (GCNII).

Rule of thumb: 2–3 layers is optimal for most tasks; rarely benefit from >5 layers without mitigation.

Scalability

Full-graph training requires all neighbours in memory. For billion-node graphs this is infeasible.

Solutions:

  • Neighbour sampling (GraphSAGE): Sample fixed-size neighbourhood
  • Cluster-GCN: Mini-batch over graph clusters
  • GraphSAINT: Sample subgraphs with importance weights
  • Lazy propagation: Precompute fixed diffusion (SGC, SIGN)
Heterogeneous & Dynamic Graphs

Heterogeneous graphs have multiple node/edge types (e.g., user, item, tag). Use type-specific transformation matrices; HAN (heterogeneous attention) uses meta-path aggregation.

Dynamic/temporal graphs have time-stamped edges. Approaches:

  • Snapshot-based: GNN per time window + RNN/Transformer over time
  • Event-based (TGN): Temporal Graph Networks maintain node memory updated on each interaction event

Multi-modal Models

Multi-modal models process and generate across multiple data types — text, images, audio, video, code. They are the frontier of AI: CLIP, GPT-4o, Gemini, Flamingo, and Stable Diffusion are all multi-modal.

CLIP — Contrastive Language-Image Pre-training

How CLIP Works (Radford et al. 2021)

Train image encoder (ViT or ResNet) and text encoder (Transformer) jointly on 400M (image, caption) pairs from the web using contrastive loss.

L = NT-Xent(image_embeds, text_embeds) — match N pairs in batch of N²

At inference: embed query text + all candidate images → pick image with highest cosine similarity. Zero-shot classification: text prompts are "a photo of a {class}".

import clip, torch model, preprocess = clip.load("ViT-B/32") image = preprocess(pil_image).unsqueeze(0) text = clip.tokenize(["a dog", "a cat", "a car"]) with torch.no_grad(): logits, _ = model(image, text) probs = logits.softmax(dim=-1) # zero-shot prediction

Architecture Patterns

Text-to-Image: Stable Diffusion Architecture

Latent Diffusion (LDM)

Stable Diffusion runs diffusion in a latent space (4×64×64) not pixel space (3×512×512) — 48× smaller, 8× faster.

  1. VAE Encoder: Compress image to latent z
  2. Forward diffusion: Add noise to z over T steps
  3. U-Net denoiser: Predict noise conditioned on CLIP text embedding via cross-attention
  4. Classifier-free guidance: 50% of training uses null text → boosted signal: ε_guided = ε_uncond + s·(ε_cond − ε_uncond)
  5. VAE Decoder: Reconstruct pixel image from denoised latent

Anomaly Detection

Anomaly detection finds data points that don't conform to expected patterns — without labelled anomaly examples in most cases. Used in fraud detection, network intrusion, manufacturing QC, and monitoring.

Methods Overview

Statistical Methods
  • Z-score: Flag |x − μ| / σ > 3. Assumes Gaussian distribution. Breaks for multivariate or heavy-tailed data.
  • IQR method: Flag x < Q1 − 1.5·IQR or x > Q3 + 1.5·IQR. Robust to skew.
  • Grubbs test: Hypothesis test for single outlier in univariate normal sample.
  • CUSUM: Cumulative sum control chart — detects small persistent shifts in mean. Used in SPC and network monitoring.
  • Mahalanobis distance: d = √((x−μ)ᵀΣ⁻¹(x−μ)) — accounts for covariance. Good for multivariate Gaussian data.
from scipy import stats z_scores = np.abs(stats.zscore(df)) outliers = (z_scores > 3).any(axis=1)

Concept Drift

The statistical relationship between features and labels changes over time — model accuracy degrades.

Types:

  • Sudden drift: Abrupt change (system update, world event)
  • Gradual drift: Slow shift in distribution over time
  • Recurring drift: Seasonal patterns (weekday vs weekend behaviour)
  • Incremental drift: Monotonic slow change (population ageing)

Detection: ADWIN (adaptive windowing), DDM, Page-Hinkley test, KSWIN. Monitor rolling accuracy or prediction distribution.

Response: Retrain on recent window, weighted training (recent examples weighted higher), ensemble with fresh model.

Worked System Design Examples

YouTube Video Recommendation

Goal: Maximise watch time while avoiding clickbait and harmful content.

Two-stage architecture:

  • Candidate generation (recall): Two-tower model embeds user + item; ANN retrieves top 500 from 500M videos. Features: watch history, search history, demographics.
  • Ranking (precision): Deep neural network scores 500 candidates; features: video age, CTR, watch percentage, user-video co-occurrence. Outputs: P(watch 30s), P(like), P(share).

Label: Weighted combination of engagement signals — watch time > likes > clicks (to avoid clickbait). Position bias correction via inverse propensity weighting.

Monitoring: Watch time, satisfaction survey, diversity metrics, fairness by creator demographics.

Fraud Detection (Real-Time)

Goal: Flag fraudulent transactions in <100ms with <0.1% false positive rate on legitimate transactions.

Constraints: Highly imbalanced (0.1% fraud), adversarial (fraudsters adapt), real-time latency requirement.

Design:

  • Features: Transaction amount, merchant category, velocity (# transactions last 1h/24h/7d from online store), distance from last transaction, device fingerprint
  • Model: GBM (interpretable, fast inference) + deep neural net ensemble. Threshold calibrated to FPR budget.
  • Real-time: Redis feature store for velocity counts; p99 latency < 50ms
  • Feedback loop: Human review queue for borderline cases; label confirmed fraud → retrain weekly

Metrics: FPR at 80% recall; dollar value of fraud caught; false decline rate.

Search Ranking

Task: Given query q and candidate documents D, rank D by relevance.

Learning to Rank:

  • Pointwise: Predict relevance score per (q,d) pair — MSE/cross-entropy
  • Pairwise: Predict which of (d₁, d₂) is more relevant for q — RankNet loss
  • Listwise: Directly optimise NDCG over the full list — LambdaMART, softmax over list

Features: BM25, PageRank, TF-IDF, click-through rate, freshness, query-document semantic similarity (BERT bi-encoder).

Ads Click-Through Rate

Task: P(click | user, ad, context) — calibrated probability, not just ranking.

Scale: Billions of predictions/day, millisecond latency.

Architecture: Sparse feature embeddings (user ID, item ID, cross features) → deep neural net → sigmoid. DLRM (Meta), Wide & Deep (Google).

Key techniques:

  • Embedding tables (10M+ categorical features)
  • FTRL-Proximal for online learning
  • Isotonic regression for calibration
  • Feedback delay handling (delayed labels)
Common Failure Modes
  • Training-serving skew: Features computed differently offline vs online
  • Feedback loops: Model predictions influence labels used for retraining (filter bubbles, popularity bias)
  • Data leakage: Future information in training features
  • Cold start: No history for new users/items
  • Label delay: Conversion labels arrive days later — stale negatives
  • Distribution shift: Training data doesn't match production
  • Adversarial gaming: Users/sellers optimise for the metric (SEO, ad fraud)

Reinforcement Learning Deep Dive

RL trains agents by trial and error — maximising cumulative reward through interaction with an environment. It powers game-playing AI, robotics, LLM alignment (RLHF), and real-time bidding.

Core Formalism: MDPs

Markov Decision Process

An MDP is a tuple (S, A, P, R, γ) where:

  • S — state space
  • A — action space
  • P(s′|s,a) — transition dynamics
  • R(s,a,s′) — reward function
  • γ ∈ [0,1) — discount factor

Goal: Find policy π*(a|s) that maximises expected discounted return:

G_t = Σ_{k=0}^∞ γᵏ R_{t+k+1}

Markov property: P(s_{t+1}|s_t,a_t) = P(s_{t+1}|s_0,...,s_t,a_t) — future depends only on current state, not history.

Value Functions

State value V^π(s): Expected return starting from s, following π:

V^π(s) = E_π[G_t | S_t=s]

Action value Q^π(s,a): Expected return taking a in s, then following π:

Q^π(s,a) = E_π[G_t | S_t=s, A_t=a]

Bellman equation:

V^π(s) = Σ_a π(a|s) Σ_{s′} P(s′|s,a)[R + γV^π(s′)]

Advantage A(s,a) = Q(s,a) − V(s) — how much better action a is vs the average. Used in actor-critic methods.

Deep Q-Networks & PPO

DQN (Mnih et al. 2015)

Approximate Q(s,a;θ) with a neural network. Two key stabilisation tricks:

  • Experience replay: Store transitions in buffer D; sample random mini-batches — breaks temporal correlations
  • Target network: Use frozen θ⁻ for TD targets; update every C steps — stabilises training
L(θ) = E[(r + γ max_{a′} Q(s′,a′;θ⁻) − Q(s,a;θ))²]

Improvements: Double DQN (decouple selection/evaluation to remove max bias), Dueling DQN (separate V and A streams), Prioritised Replay (sample high-TD-error transitions more), Rainbow (all combined).

from stable_baselines3 import DQN model = DQN("MlpPolicy", "CartPole-v1", learning_rate=1e-3, buffer_size=50000, exploration_fraction=0.1, verbose=1) model.learn(total_timesteps=100_000)
PPO (Proximal Policy Optimisation)

The workhorse algorithm for continuous control and LLM alignment. Constrains policy update size to prevent catastrophic collapse.

Clipped objective (avoids trust-region penalty):

L^CLIP = E[min(r_t(θ)Â_t, clip(r_t(θ), 1−ε, 1+ε)Â_t)]

where r_t(θ) = π_θ(a|s) / π_θ_old(a|s) is the probability ratio, Â_t is the advantage estimate, and ε=0.2 is the clip range.

Full PPO loss: L = L^CLIP − c₁·L^VF + c₂·H[π_θ] (value loss + entropy bonus for exploration)

SAC, Offline RL & Multi-Agent

Soft Actor-Critic (SAC)

Maximum entropy RL — augments reward with policy entropy to encourage exploration:

J(π) = Σ_t E[r_t + α·H(π(·|s_t))]

α (temperature) balances exploration vs exploitation — auto-tuned by constraining expected entropy to a target value.

Key features:

  • Off-policy (efficient replay buffer use)
  • Clipped double-Q trick (like TD3) to reduce overestimation
  • Reparameterisation trick for continuous actions
  • State-of-the-art for continuous control (MuJoCo)
Offline RL

Learn from a fixed logged dataset — no environment interaction. Critical when environment is expensive or dangerous (healthcare, robotics).

Core challenge: Distributional shift — Q-values are extrapolated for out-of-distribution actions not in the dataset → overestimation → policy fails.

Model-Based & Multi-Agent RL

Model-based RL: Learn a world model P(s′,r|s,a); plan inside it (Dyna-Q) or use it for imagination rollouts (Dreamer, MBPO). Far more sample-efficient but sensitive to model errors (compounding).

MuZero: Learns a latent world model + value + policy; no knowledge of game rules needed. Beats AlphaZero on Atari + chess simultaneously.

  • MADDPG: Centralised critic (sees all agents) + decentralised actors
  • QMIX: Monotonic mixing network for cooperative tasks
  • MAPPO: PPO with centralised value function — strong baseline for cooperative MARL

RL in Practice: Common Pitfalls

Reward Hacking & Pitfalls
  • Reward hacking: Agent finds unintended ways to maximise reward (boat racing agent spinning in circles for score)
  • Sparse rewards: Agent never reaches positive reward during random exploration. Fix: reward shaping, curriculum learning, hindsight experience replay (HER)
  • Catastrophic forgetting: Policy improves on new tasks but forgets old ones. Fix: EWC, replay buffers, population-based training
  • Sample inefficiency: On-policy methods (PPO) need millions of steps. Fix: off-policy (SAC), model-based, offline RL
  • Hyper-sensitivity: RL results vary enormously with random seed — always report mean ± std over ≥ 5 seeds
  • Sim-to-real gap: Policy trained in simulation fails in real world. Fix: domain randomisation, real-world fine-tuning
RL for LLMs (RLHF & Beyond)

RLHF recap: SFT → Reward Model → PPO. Policy = LLM, action = next token, reward = RM score at end of sequence, KL penalty keeps it close to SFT reference.

GRPO (Group Relative Policy Optimisation): Used in DeepSeek-R1. Removes value network — estimates baseline from group of sampled outputs. Simpler and more memory-efficient than PPO.

Constitutional AI (CAI): Use AI feedback instead of human feedback — model critiques own outputs using a constitution of principles, then revises. Scales without human labelers.

Process Reward Models (PRMs): Reward each reasoning step (not just final answer) — better for math and chain-of-thought tasks. Used in OpenAI o1.

Optimization Theory

Understanding the mathematical underpinnings of training — convexity, loss landscapes, weight initialisation, and convergence — separates practitioners who tune blindly from those who diagnose and fix systematically.

Convex vs Non-Convex Optimisation

Convexity

A function f is convex if for all x, y and λ ∈ [0,1]:

f(λx + (1−λ)y) ≤ λf(x) + (1−λ)f(y)

Convex problems: Any local minimum is a global minimum. GD converges (with appropriate LR). Examples: linear regression (MSE), logistic regression, SVM.

Strongly convex (f ≽ μI): Unique minimum; GD converges linearly — error decays as O((1−μ/L)ᵏ) where L is Lipschitz constant of gradient.

Deep networks are non-convex — but empirically still trainable due to overparameterisation, loss landscape structure, and implicit regularisation of SGD.

Loss Landscape

Saddle points: Gradient = 0 but not a minimum — positive curvature in some directions, negative in others. Common in high-dimensional non-convex losses. SGD noise helps escape them.

Sharp vs flat minima: Flat minima generalise better — small perturbations to weights barely change the loss. Sharp minima overfit to training data.

SAM (Sharpness-Aware Minimisation): Explicitly seeks flat minima by perturbing weights in worst direction and minimising that perturbed loss:

min_θ max_{‖ε‖≤ρ} L(θ+ε)

Mode connectivity: Two independently trained minima are typically connected by a low-loss path in weight space — the loss landscape is not isolated valleys but a connected manifold.

KKT Conditions

Karush-Kuhn-Tucker conditions — necessary (and sufficient for convex problems) for constrained optimisation:

For min f(x) s.t. gᵢ(x) ≤ 0, hⱼ(x) = 0:

  • Stationarity: ∇f + Σλᵢ∇gᵢ + Σμⱼ∇hⱼ = 0
  • Primal feasibility: gᵢ(x) ≤ 0, hⱼ(x) = 0
  • Dual feasibility: λᵢ ≥ 0
  • Complementary slackness: λᵢgᵢ(x) = 0

SVM derivation: SVM is a QP with KKT conditions — support vectors are those where λᵢ > 0 (complementary slackness active). Dual formulation leads to kernel trick.

Weight Initialisation

Why Initialisation Matters

Vanishing/exploding gradients: In a deep network with L layers, gradients are products of L Jacobians. If each matrix has singular values <1, gradient → 0 (vanishing). If >1, gradient → ∞ (exploding).

Symmetry breaking: All weights = 0 → all neurons identical → no learning. Must break symmetry with random initialisation.

Xavier / He Derivation

Goal: Keep variance of activations and gradients stable across layers.

For a linear layer y = Wx with n_in inputs, if W ~ N(0, σ²) and x has unit variance:

Var(y) = n_in · σ² · Var(x)

To keep Var(y) = Var(x): σ² = 1/n_in (He for ReLU, which kills ~half variance, so σ² = 2/n_in). Xavier averages forward and backward: σ² = 2/(n_in + n_out).

Convergence Theory

SGD convergence (non-convex): Under smoothness (L-Lipschitz gradient), SGD finds an ε-stationary point (‖∇L‖ ≤ ε) in O(1/ε²) iterations.

Implicit regularisation: SGD with small batches implicitly prefers flatter minima — the gradient noise acts as a regulariser that biases towards solutions with better generalisation.

Catapult phase: At the start of training with large LR, loss initially spikes before dropping — the network is being pushed out of a sharp basin into a flatter region.

Neural tangent kernel (NTK): In the infinite-width limit, gradient descent on neural networks is equivalent to kernel regression with the NTK — provides theoretical insight into convergence but doesn't explain practical finite-width training well.

Grokking: Models sometimes generalise long after memorisation — continued training eventually finds a structured solution even after near-zero training loss.

Normalisation Layers: Theory

BatchNorm

Normalise each feature across the batch; learnable γ, β restore expressiveness:

x̂ = (x − μ_B)/√(σ²_B+ε)  →  γx̂ + β

Why it works: Reduces internal covariate shift; allows higher LR; acts as regulariser (noise from batch statistics).

Problems: Batch size dependent; different behaviour train vs inference; doesn't work with batch size 1; problematic for RNNs and variable-length sequences.

Running stats: At inference uses EMA of μ, σ² from training — must set model.eval().

LayerNorm & RMSNorm

LayerNorm: Normalise across feature dimension (not batch) — independent per sample. Works with any batch size; standard in Transformers.

LN(x) = γ·(x − μ)/√(σ²+ε) + β  (μ,σ over features)

RMSNorm (simpler, used in Llama/Mistral): Drop the mean subtraction — just scale by RMS:

RMSNorm(x) = γ · x / √(mean(x²)+ε)

10–20% faster than LayerNorm; empirically matches or exceeds quality in large LLMs.

GroupNorm & When to Use What

GroupNorm: Divide channels into G groups; normalise within each group. Batch-size independent; used in detection/segmentation where batch size is small (1–2 images).

InstanceNorm: Normalise each sample × channel independently. Used in style transfer — removes content statistics, preserves style.

Healthcare ML Pitfalls
  • Shortcut learning: Models learn hospital-specific artefacts (scanner brand, demographic mix) rather than pathology
  • Label noise: ICD codes are billing codes, not ground truth diagnoses — systematic biases
  • Temporal leakage: Using notes written after the event you're predicting (discharge summary to predict admission outcome)
  • Demographic bias: Pulse oximetry less accurate for dark skin; models trained on majority populations fail on minorities
  • Uncertainty: Calibrated confidence is critical — model must express "I don't know" reliably
  • Human-in-loop: Most clinical AI is decision support, not autonomous — design for appropriate reliance

Finance & Quantitative ML

Time Series in Finance

Backtesting pitfalls — the most dangerous part:

  • Lookahead bias: Using data available only after the trade decision (e.g., daily close to make intraday decision)
  • Survivorship bias: Only modelling stocks still listed — ignores delistings and bankruptcy
  • Overfitting: Sharpe ratio inflates with number of parameters; use combinatorial purged cross-validation (CPCV)
  • Transaction costs: Slippage, spread, market impact — often wipe out apparent alpha entirely

Evaluation metrics:

  • Sharpe ratio: (μ_p − r_f) / σ_p — risk-adjusted return
  • Max drawdown: Largest peak-to-trough decline
  • Calmar ratio: CAGR / max drawdown
  • Information ratio: Active return / tracking error

Autonomous Vehicles & Robotics

Perception Stack

Sensors: Camera (rich texture, cheap), LiDAR (accurate 3D, expensive), Radar (all weather, velocity), GPS+IMU (localisation).

Object detection: BEV (Bird's Eye View) representations — project all sensor data to top-down map. BEVFusion fuses camera + LiDAR in BEV space.

3D detection: PointPillars, VoxelNet, CenterPoint — process LiDAR point clouds as voxels or pillars, detect 3D bounding boxes.

Tracking: Multi-object tracking (MOT) via Kalman filter + Hungarian algorithm for data association across frames.

Planning & Control

HD Maps: Pre-computed centimetre-accurate maps with lane boundaries, speed limits, traffic lights — autonomous driving relies heavily on them; Tesla notably avoids them.

Prediction: Predict future trajectories of other agents — multimodal (multiple plausible futures). Models: TNT, MTR, Wayformer (Transformer over agent history).

Planning: Rule-based planners (Waymo early), learned planners (imitation learning + RL), occupancy grid networks.

End-to-end: UniAD, Tesla FSD — single neural network from sensors to control outputs. Harder to debug but potentially more integrated.

Safety & Validation

Long tail problem: 99.9% of driving is easy; the remaining 0.1% edge cases (jaywalkers, construction, weather) determine safety. Requires billions of miles of data or simulation.

Simulation: Carla, SUMO, Waymo's simulation environment — replay logged scenarios with synthetic perturbations. Cannot fully capture sim-to-real gap.

ISO 26262: Functional safety standard for road vehicles — requires ASIL (Automotive Safety Integrity Level) classification and fault tree analysis.

Disengagement rate: Industry metric — how often human must take over per mile. Not standardised across companies, making comparison misleading.

Backpropagation from Scratch

2-Layer MLP + Backprop
import numpy as np def sigmoid(x): return 1 / (1 + np.exp(-x)) class MLP: def __init__(self, n_in, n_h, n_out): self.W1 = np.random.randn(n_in, n_h) * np.sqrt(2/n_in) self.b1 = np.zeros((1, n_h)) self.W2 = np.random.randn(n_h, n_out) * np.sqrt(2/n_h) self.b2 = np.zeros((1, n_out)) def forward(self, X): self.X = X self.z1 = X @ self.W1 + self.b1 self.a1 = np.maximum(0, self.z1) # ReLU self.z2 = self.a1 @ self.W2 + self.b2 self.a2 = sigmoid(self.z2) # output return self.a2 def backward(self, y, lr=0.01): m = y.shape[0] dL_da2 = self.a2 - y # BCE grad dL_dW2 = self.a1.T @ dL_da2 / m dL_db2 = dL_da2.mean(axis=0, keepdims=True) dL_da1 = dL_da2 @ self.W2.T dL_dz1 = dL_da1 * (self.z1 > 0) # ReLU grad dL_dW1 = self.X.T @ dL_dz1 / m dL_db1 = dL_dz1.mean(axis=0, keepdims=True) self.W2 -= lr * dL_dW2; self.b2 -= lr * dL_db2 self.W1 -= lr * dL_dW1; self.b1 -= lr * dL_db1
Scaled Dot-Product Attention
import numpy as np def softmax(x, axis=-1): x = x - x.max(axis=axis, keepdims=True) e = np.exp(x) return e / e.sum(axis=axis, keepdims=True) def attention(Q, K, V, mask=None): """ Q,K,V: (batch, heads, seq, d_k) Returns: (batch, heads, seq, d_v) """ d_k = Q.shape[-1] scores = Q @ K.swapaxes(-2, -1) / np.sqrt(d_k) if mask is not None: scores = np.where(mask, -1e9, scores) attn = softmax(scores) # (batch, heads, seq, seq) return attn @ V, attn # Causal mask for GPT-style autoregressive attention seq = 4 causal_mask = np.triu(np.ones((seq, seq), dtype=bool), k=1)

K-Means Clustering
import numpy as np class KMeans: def __init__(self, k, max_iter=100): self.k = k; self.max_iter = max_iter def fit(self, X): idx = np.random.choice(len(X), self.k, replace=False) self.centroids = X[idx] for _ in range(self.max_iter): labels = self._assign(X) new_c = np.array([X[labels==j].mean(0) for j in range(self.k)]) if np.allclose(new_c, self.centroids): break self.centroids = new_c return self def _assign(self, X): dists = np.linalg.norm( X[:, None] - self.centroids[None], axis=-1) return dists.argmin(axis=1) def predict(self, X): return self._assign(X)
Decision Tree (CART)
import numpy as np def gini(y): classes, counts = np.unique(y, return_counts=True) p = counts / len(y) return 1 - (p**2).sum() def best_split(X, y): best = {'gain': -1} parent_gini = gini(y) for feat in range(X.shape[1]): for thresh in np.unique(X[:, feat]): left = y[X[:, feat] <= thresh] right = y[X[:, feat] > thresh] if len(left)==0 or len(right)==0: continue n = len(y) gain = parent_gini - ( len(left)/n*gini(left) + len(right)/n*gini(right)) if gain > best['gain']: best = {'gain':gain, 'feat':feat, 'thresh':thresh} return best

Linear Regression: Normal Equation & Gradient Descent
import numpy as np # Closed-form: θ = (XᵀX)⁻¹Xᵀy def linear_reg_ols(X, y): X = np.column_stack([np.ones(len(X)), X]) return np.linalg.lstsq(X, y, rcond=None)[0] # Gradient descent def linear_reg_gd(X, y, lr=0.01, epochs=1000): m, n = X.shape theta = np.zeros(n + 1) X_ = np.column_stack([np.ones(m), X]) for _ in range(epochs): grad = X_.T @ (X_ @ theta - y) / m theta -= lr * grad return theta # Ridge regression: θ = (XᵀX + λI)⁻¹Xᵀy def ridge(X, y, lam=1.0): X_ = np.column_stack([np.ones(len(X)), X]) I = np.eye(X_.shape[1]); I[0,0] = 0 # don't penalise bias return np.linalg.solve(X_.T @ X_ + lam * I, X_.T @ y)
PCA from Scratch
import numpy as np class PCA: def __init__(self, n_components): self.n = n_components def fit(self, X): self.mean = X.mean(axis=0) X_c = X - self.mean # Covariance via SVD (numerically stable) U, S, Vt = np.linalg.svd(X_c, full_matrices=False) self.components = Vt[:self.n] # top n eigenvectors self.explained_var = (S**2) / (len(X)-1) return self def transform(self, X): return (X - self.mean) @ self.components.T def inverse_transform(self, Z): return Z @ self.components + self.mean # Explained variance ratio # pca.explained_var[:n] / pca.explained_var.sum()
