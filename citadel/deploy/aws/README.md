# CITADEL — AWS (Commercial) Deployment Runbook

**CITADEL** — *Code Inspection, Threat Analysis & Deployment Evaluation Lab* — is a
source-code and executable security & compliance review platform. This package
deploys the **production** CITADEL **deep-scan backend** (a Node 20 Express API
that runs real open-source scanners — Semgrep, Bandit, Trivy, Syft, Grype,
Gitleaks, Checkov, OSV-Scanner, Hadolint, ClamAV — and serves the CITADEL SPA)
to **commercial AWS** (standard partition `aws`).

> Partition: `aws` · Default region: `us-east-1` (any commercial region works).
> Standard (non-FIPS) service endpoints by default — flip `use_fips_endpoint` if
> your compliance scope requires FIPS 140-3 endpoints. Data-in-transit uses
> **TLS 1.2/1.3** at the ALB and **TLS (sslmode=require)** to PostgreSQL;
> data-at-rest uses **KMS CMKs** (ECR, Secrets Manager, RDS, CloudWatch Logs).

> Looking for the GovCloud / FedRAMP-High variant? See `../aws-gov/`. **Do not**
> mix the two — this stack rejects `us-gov-*` and `cn-*` regions by validation.

## 1. Overview & Target Architecture

The container runs on **ECS Fargate** in **private subnets**, fronted by an
**Application Load Balancer** protected by **AWS WAFv2**. Images live in **ECR**
(scan-on-push, immutable tags). Durable state lives in **RDS PostgreSQL**
(private, Multi-AZ, KMS-encrypted) — this is what lets CITADEL run multiple
Fargate tasks and (optionally) multi-tenant. All secret-class values
(`CITADEL_JWT_SECRET`, `CITADEL_ADMIN_PASSWORD`, `CITADEL_SUPERADMIN_TOKEN`,
`CITADEL_METRICS_TOKEN`, and the assembled `DATABASE_URL`) are **generated by
Terraform** into **Secrets Manager** (CMK-encrypted) and injected into the task
as `secrets` (valueFrom) — never as plaintext. Logs flow to **CloudWatch Logs**
(KMS-encrypted) plus **VPC Flow Logs**.

Private subnets reach AWS APIs and scanner vuln-DB mirrors (Trivy/Grype/ClamAV
updates) through **NAT gateways** (one per AZ for HA); an **S3 gateway endpoint**
keeps ECR layer pulls off NAT. Tasks have **no public IP**.

## 2. Prerequisites

| Requirement | Detail |
|---|---|
| AWS account | Commercial partition `aws`. Verify with `aws sts get-caller-identity` (ARN should contain `:aws:`, not `aws-us-gov`). |
| IAM principal | Permissions for ECS, ECR, ELBv2, WAFv2, KMS, Secrets Manager, RDS, CloudWatch, IAM, VPC. Use a deploy role, not long-lived root keys. |
| AWS CLI v2 | Configured with commercial credentials for the target region. |
| Terraform | >= 1.5, AWS provider `~> 5.40`, random provider `~> 3.6`. |
| Docker | For building/pushing the image (`linux/amd64` to match Fargate). |
| ACM certificate | A certificate in the **same region** for the HTTPS listener (`acm_certificate_arn` / `ACM_CERT_ARN`). |

Commercial-AWS notes:
- **ECR registry host:** `<account-id>.dkr.ecr.<region>.amazonaws.com`.
- **ARNs** use the `aws` partition; the Terraform builds these partition-aware
  via `data.aws_partition.current`, so the same code works in any commercial region.
- The image **must be built from the repository root** with the server Dockerfile
  (`docker build -f citadel/server/Dockerfile -t citadel-server .`) so both the
  SPA (`citadel/`) and backend (`citadel/server/`) are in the build context.

## 3. Step-by-Step Deployment

### 3.1 One-command path

```bash
cd citadel/deploy/aws
export REGION=us-east-1
export ACM_CERT_ARN=arn:aws:acm:us-east-1:<acct>:certificate/<id>

./deploy.sh
```

`deploy.sh` resolves the account/partition, ensures the ECR repo exists, logs in
to ECR, builds and pushes the image **from the repo root**, then runs
`terraform init/plan/apply` and prints the service URL.

### 3.2 Manual path (for review / change control)

**Verify you are in commercial AWS:**
```bash
aws sts get-caller-identity --query 'Arn' --output text   # ARN contains :aws: (not aws-us-gov)
```

**Authenticate Docker to ECR:**
```bash
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
REGION=us-east-1
REGISTRY=${ACCOUNT_ID}.dkr.ecr.${REGION}.amazonaws.com

aws ecr get-login-password --region ${REGION} \
  | docker login --username AWS --password-stdin ${REGISTRY}
```

**Build & push (from the repo root, linux/amd64):**
```bash
# Run from the repository root so the build context includes citadel/ + citadel/server/
TAG=$(git rev-parse --short HEAD)
docker build --platform linux/amd64 \
  -f citadel/server/Dockerfile \
  -t ${REGISTRY}/citadel-prod:${TAG} .
docker push ${REGISTRY}/citadel-prod:${TAG}
```

**Provision infrastructure:**
```bash
cd citadel/deploy/aws
terraform init
terraform plan  -var "region=us-east-1" -var "image_tag=${TAG}" \
                -var "acm_certificate_arn=${ACM_CERT_ARN}" -out tfplan
terraform apply tfplan
terraform output service_url
```

**Retrieve the generated bootstrap admin password** (set `CITADEL_ADMIN_EMAIL`
via `-var citadel_admin_email=...`; the password is generated for you):
```bash
aws secretsmanager get-secret-value --region us-east-1 \
  --secret-id citadel-prod/admin-password \
  --query SecretString --output text
```

**Restrict ingress for production** — set `ingress_allowed_cidrs` to your office /
VPN ranges instead of `0.0.0.0/0`, and re-apply.

### 3.3 Multi-tenant mode

Set `-var citadel_multitenant=true -var citadel_base_domain=citadel.example.com`.
The toggle sets `CITADEL_MULTITENANT=1` and `CITADEL_BASE_DOMAIN` in the task;
RDS provides the shared, durable store across tasks/tenants.

## 4. Secrets & Configuration

| Variable | Type | Source | Notes |
|---|---|---|---|
| `DATABASE_URL` | secret | Secrets Manager | Assembled `postgresql://…?sslmode=require` from the RDS endpoint + generated password. |
| `CITADEL_JWT_SECRET` | secret | Secrets Manager | 64-char random, generated by Terraform. |
| `CITADEL_ADMIN_PASSWORD` | secret | Secrets Manager | Random; retrieve to log in, then rotate. |
| `CITADEL_SUPERADMIN_TOKEN` | secret | Secrets Manager | 48-char random. |
| `CITADEL_METRICS_TOKEN` | secret | Secrets Manager | 48-char random. |
| `CITADEL_ADMIN_EMAIL` | env | `var.citadel_admin_email` | Non-secret. |
| `CITADEL_MULTITENANT` | env | `var.citadel_multitenant` | `1`/`0`. |
| `CITADEL_BASE_DOMAIN` | env | `var.citadel_base_domain` | Tenant routing base. |
| `NODE_ENV` | env | static | `production`. |
| `PGSSL` | env | static | `1` — pairs with `sslmode=require` and RDS `force_ssl`. |
| `PORT` | env | `var.container_port` | `8080`. |
| `CITADEL_TMP` | env | `var.scratch_mount_path` | `/tmp/citadel`, mounted as the ephemeral scratch volume. |

Secret-class values are GENERATED (`random_password`) — there are **no hardcoded
secrets** in this stack. To rotate, change the secret value in Secrets Manager
(or taint the `random_password` resource) and force a new deployment.

## 5. Security & Compliance Hardening Checklist

Mapped to **NIST SP 800-53 Rev5** control families. Use as deployment evidence.

- [ ] **AC-3 / AC-6 (CM-7) — Least privilege.** Task role grants only ECS-Exec
      messaging; execution role only ECR pull (repo-scoped), logs, and the
      specific secret ARNs. Container runs non-root (UID 10001), read-only root
      FS, all caps dropped.
- [ ] **AC-4 / SC-7 — Boundary protection.** Tasks in private subnets, no public
      IP; only the ALB is internet-facing; SGs are least-privilege (ALB→ECS→RDS
      chain). Egress via per-AZ NAT. Set `ingress_allowed_cidrs` to approved ranges.
- [ ] **AU-2 / AU-6 / AU-12 — Audit logging.** CloudWatch Logs (app), VPC Flow
      Logs, RDS log exports, Container Insights, WAF logging.
- [ ] **AU-9 / AU-11 — Audit protection & retention.** Logs KMS-encrypted;
      retention `var.log_retention_days` (default 90).
- [ ] **CM-2 / CM-6 — Baseline & config.** Infra is Terraform (versioned);
      immutable ECR tags; pin image digests in production.
- [ ] **CP-9 — Backups.** RDS automated backups (`db_backup_retention_days`),
      Multi-AZ failover, final snapshot on destroy.
- [ ] **IA-2 / IA-5 — Identification & authenticators.** No static creds in image
      or state; secrets generated into Secrets Manager (CMK); injected via task `secrets`.
- [ ] **RA-5 / SI-2 — Vuln scanning & flaw remediation.** ECR scan-on-push;
      remediate Critical/High before promotion.
- [ ] **SC-7 — WAF.** WAFv2 with AWS managed CommonRuleSet, KnownBadInputs, SQLi,
      and IP rate-limiting associated to the ALB.
- [ ] **SC-8 / SC-13 — Transit protection.** HTTP→HTTPS redirect; HTTPS listener
      with a modern TLS1.2/1.3 policy; Postgres `force_ssl=1` + `sslmode=require`
      + app `PGSSL=1`. Optional FIPS endpoints via `use_fips_endpoint`.
- [ ] **SC-12 / SC-28 — Key mgmt & rest encryption.** Customer-managed KMS key
      with rotation; ECR, Secrets Manager, RDS, and Logs all CMK-encrypted.

## 6. Logging, Monitoring, Backup & DR

**Logging / Monitoring**
- CloudWatch Logs: `/citadel/prod/app` and `/citadel/prod/vpc-flow` (KMS-encrypted).
- RDS log exports (`postgresql`, `upgrade`) + Performance Insights (CMK-encrypted).
- WAF metrics + sampled requests in CloudWatch; alarm on rate-limit blocks and
  5xx surges. Container Insights for task-level CPU/memory.
- Consider enabling CloudTrail (org trail), GuardDuty, and Security Hub
  account-wide — out of scope for this stack but recommended.

**Backup**
- RDS: automated backups + Multi-AZ; a final snapshot is taken on destroy. Add a
  read replica / cross-region snapshot copy for DR if required.
- Terraform remote state: S3 (SSE-KMS, versioning) + DynamoDB lock — see the
  commented `backend "s3"` block in `versions.tf`.
- ECR images are durable; keep last-known-good tags per the lifecycle policy.

**Disaster Recovery**
- Infra is fully codified — re-`terraform apply` in another region to rebuild
  (RTO driven by RDS snapshot restore + ECR image availability + an ACM cert in
  the DR region). Run `terraform plan` regularly to detect drift.

## 7. Cost Notes

Primary cost drivers:
- **ECS Fargate**: `desired_count` × (`task_cpu`/`task_memory`) per second.
  Default 2 × (2 vCPU / 4 GB) for the scanner toolchain + HA. Scale to 1 for non-prod.
- **RDS**: instance-hours (`db_instance_class`, doubled for Multi-AZ) + storage +
  backups. Default `db.t3.medium` Multi-AZ.
- **NAT gateways**: hourly per-AZ + data processing. **ALB**: hourly + LCU.
  **WAFv2**: per web ACL + per rule + per request.
- **KMS**: per CMK/month + per request. **CloudWatch Logs**: ingestion + storage.

Cost-saving levers: set `desired_count=1`, `db_multi_az=false`, disable Container
Insights, and shorten log retention in non-prod. A single NAT can be used in
non-prod by reducing public subnet count (one AZ).

## 8. Teardown

ALB deletion protection and RDS deletion protection are on by default and must be
disabled before `terraform destroy`:

```bash
cd citadel/deploy/aws

# Disable protections so destroy succeeds (or re-apply with the vars set false):
terraform apply \
  -var "enable_deletion_protection=false" \
  -var "db_deletion_protection=false" -auto-approve

terraform destroy -var "region=us-east-1"
```

> Notes:
> - The KMS CMK has a 30-day deletion window; it schedules deletion rather than
>   removing immediately.
> - RDS takes a **final snapshot** (`citadel-prod-pg-final`) on destroy unless you
>   set `skip_final_snapshot = true`. Delete the snapshot manually if not needed.
> - ECR has `force_delete = false`; remove images first if the repo is non-empty.
