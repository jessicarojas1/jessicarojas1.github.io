Overview

CITADEL (Code Inspection, Threat Analysis & Deployment Evaluation Lab) is a source-code and executable review platform. The public demo runs entirely in your browser — uploaded files are parsed locally and never transmitted — while the production build packages the same front-end with an optional API/worker tier that orchestrates heavyweight open-source scanners (Semgrep, Trivy, Syft/Grype, Gitleaks, ClamAV, Bandit) for deployment to Azure Government or AWS GovCloud.

It answers, in one report: How secure is this code? How well is it written? What languages does it use? How is it deployed? Which compliance obligations does it touch?

Architecture

The browser app is a layered pipeline. Each stage is an independent module attached to a global CITADEL namespace and communicates through plain data objects — no framework lock-in.

Data flow

  1. Intake. app.js collects File objects from drop, picker, or folder selection and hands them to the Ingest Engine.
  2. Normalize. ingest.js expands ZIP/JAR/APK archives via JSZip, skips dependency/build dirs, sniffs each file as text or binary, and emits a flat list of entries { path, lang, content|bytes }.
  3. Fan-out. scanner.js streams every entry to the analyzers and concatenates their findings.
  4. Score. The orchestrator computes language stats, a maintainability index, a severity-weighted security score, and an A–F grade.
  5. Map. Findings are cross-walked to control identifiers across every framework by the Compliance Mapping Engine.
  6. Render. report.js paints the scorecard, charts, finding cards, compliance posture, SBOM, and export artifacts.

Engine modules

How the modules interact

Every analyzer consumes the same normalized entry and produces objects in one shared finding shape:

finding = {
  ruleId, name, category, severity,      // what & how bad
  cwe, confidence,                        // weakness id & certainty
  file, line, snippet,                    // where
  remediation                             // how to fix
}

This single contract is what lets independent modules cooperate without coupling:

  • The SAST, Secrets, SBOM, and Binary modules each emit findings into one stream — the orchestrator never needs to know how a finding was produced.
  • The Scoring Engine only reads severity, so adding a new analyzer automatically affects the grade.
  • The Compliance Mapping Engine only reads category; a finding's framework cross-walk is derived purely from its category, so one new rule instantly lights up the right controls across all 20+ standards.
  • The Report Engine renders by severity and category and is otherwise analyzer-agnostic.

Adding coverage is therefore additive: drop a rule into rules.js with a category, and scoring, compliance mapping, charts, and exports all update with no other changes.

Production deployment

The deploy/ directory ships hardened, government-ready Infrastructure-as-Code:

  • Azure Government — Bicep + az runbook: Container Apps, ACR, Key Vault, private Storage, Log Analytics, managed identity, and an NIST 800-53 / CMMC hardening checklist. See deploy/azure-gov.
  • AWS GovCloud (US) — Terraform + aws runbook: ECS Fargate, ALB + WAF, ECR scan-on-push, KMS, Secrets Manager, S3 Object-Lock quarantine, GuardDuty/Security Hub. See deploy/aws-gov.

Both ship a multi-stage hardened Dockerfile (non-root, FIPS-friendly base, strict security-headers Nginx) so the analyzer runs inside a CUI / FedRAMP-High boundary with TLS 1.2+ FIPS, private networking, and no public ingress except the WAF.

Scope & limitations

The in-browser engine is a heuristic, pattern-based analyzer optimized for fast triage, education, and demonstration. It is intentionally transparent and dependency-light. It is not a replacement for a full data-flow SAST, DAST, IAST, or a live CVE feed. For an Authorization to Operate (ATO) package, run the production tier's integrated scanners and a credentialed vulnerability assessment, and have results reviewed by a qualified assessor. CITADEL's value is consolidating many signals into one framework-aware report and getting you to the right controls quickly.
