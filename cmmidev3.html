📚 About This Reference — CMMI v2.0 DEV ML3

CMMI v2.0 (2018) organizes practices into Practice Areas and Practice Groups. This reference covers all 19 Practice Areas required for a Development ML3 appraisal — 7 ML2 Core PAs, 4 ML2 Dev PAs (incl. SAM), 7 ML3 Core PAs, and 4 ML3 Dev PAs — each enriched with the CMMI practice statement, elaboration, and concrete compliance examples.

ML2 Core (7): CM, GOV, II, MC, MPM, PLAN, PQA — project management and discipline fundamentals required at every level.

ML2 Development (2+1): RDM (requirements), SAM (supplier agreements — optional but frequently in-scope).

ML3 Core (7): CAR, DAR, EST, OT, PAD, PCM, RSK — organizational process definition, training, risk, estimation, and improvement.

ML3 Development (4): TS (technical solution), PI (product integration), VV (verification & validation), PR (peer reviews) — the engineering lifecycle chain appraisers scrutinize most.

🔥 What Appraisers Will Actually Drill You On — CMMI DEV ML3
🧠 Engineering Lifecycle — MAKE OR BREAK
RDM → TS → PI → VV → PR
Must be fully traceable and real. This chain is the entire point of the Development appraisal.
📊 Control & Discipline
PLAN, MC, MPM, RSK
Must show you are in control of your projects — not just doing work, but managing it.
🏢 Organizational Maturity
PAD, PCM, OT, PQA
Must show this isn't a one-off project — it's institutionalized across the organization.
🔍 Decision & Improvement
DAR, CAR
Must show you think and improve like an engineering org — data-driven decisions, root cause fixes.
🚨 You will FAIL if:
  • Processes exist but aren't actually used
  • No traceability — biggest failure point
  • No real peer reviews (just "LGTM" culture)
  • No metrics tied to actual decisions
  • No root cause analysis on defects
✅ You will PASS if:
  • Everything is connected — requirements trace to tests
  • Everything is repeatable — processes work on every project
  • Everything is evidenced in real work artifacts
