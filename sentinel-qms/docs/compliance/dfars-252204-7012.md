# DFARS 252.204-7012 Coverage

**DFARS 252.204-7012 — Safeguarding Covered Defense Information and Cyber Incident Reporting** requires
contractors handling **Covered Defense Information (CDI/CUI)** to (a) provide *adequate security* by
implementing **NIST SP 800-171**, (b) **report cyber incidents within 72 hours**, (c) meet cloud
requirements, and (d) flow the clause down to subcontractors. This document maps Sentinel QMS support to
each paragraph of the clause.

> Sentinel QMS provides the **technical safeguards and evidence**. Incident reporting to DoD, contractual
> flow-downs, and the System Security Plan remain organizational obligations.

---

## 1. Adequate Security — NIST SP 800-171 (¶(b))

The clause requires implementation of all applicable 800-171 requirements. Sentinel QMS implements the
technical controls across all 14 families; see the full control mapping in
[nist-800-171-mapping.md](nist-800-171-mapping.md). Highlights:

| Requirement area | Platform implementation |
|------------------|-------------------------|
| Access control (3.1) | RBAC, federation, least privilege, deny-by-default |
| Audit & accountability (3.3) | Append-only audit log → SIEM |
| Identification & authentication (3.5) | OIDC/SAML/CAC-PIV, MFA, bcrypt |
| System & comms protection (3.13) | TLS 1.2+/FIPS, segmentation, KMS/Key Vault, encryption at rest |
| System & information integrity (3.14) | Patching, image/dependency scanning, monitoring |

---

## 2. Cloud Computing Requirements (¶(b)(2))

When using an external cloud service provider to store/process/transmit CDI, the clause requires the CSP
to meet the **FedRAMP Moderate** baseline and to comply with paragraphs (c)–(g) of the clause.

| Requirement | Implementation |
|-------------|----------------|
| FedRAMP Moderate (or equivalent) CSP | **AWS GovCloud (US)** and **Azure Government** are FedRAMP-authorized at Moderate/High and DoD-impact-level aligned |
| U.S. data residency | Region pinning to GovCloud / Azure Gov (see [itar-ear-export-control.md](itar-ear-export-control.md)) |
| Comply with ¶(c)–(g) | Incident reporting, malicious software handling, media preservation, and access for forensics supported below |

---

## 3. Cyber Incident Reporting — 72 Hours (¶(c))

The contractor must **rapidly report** (within 72 hours of discovery) a cyber incident affecting CDI or
the contractor's ability to perform, via DIBNet (https://dibnet.dod.mil), and must have a **DoD-approved
medium assurance certificate**.

| Capability | Implementation |
|------------|----------------|
| Detection | SIEM (Security Hub + GuardDuty / Sentinel + Defender) with alerting on anomalous access, auth failures, and integrity events |
| Investigation data | Immutable audit log + correlated app/infra logs provide actor/action/entity/IP/time |
| 72-hour workflow | Operations runbook defines the incident-response timeline, severity triage, and the DIBNet reporting step with required data elements |
| Required report fields | Audit/SIEM data supplies many DFARS report elements (affected systems, user accounts, technique indicators) |

See the incident-response procedure in
[../deployment/operations-runbook.md](../deployment/operations-runbook.md).

---

## 4. Malicious Software (¶(d))

If malicious software is discovered and isolated in connection with a reported incident, it must be
submitted to the DoD Cyber Crime Center (DC3).

| Capability | Implementation |
|------------|----------------|
| Isolation | Container/image scanning + EDR (GuardDuty/Defender) detect and help isolate malicious code |
| Preservation | Suspected artifacts preserved per runbook for DC3 submission (organizational step) |

---

## 5. Media Preservation & Protection (¶(e))

Preserve and protect images of affected systems and relevant monitoring/packet-capture data for **at
least 90 days** from incident report.

| Capability | Implementation |
|------------|----------------|
| Image/log preservation | DB snapshots, container images (ECR/ACR), and SIEM logs retained ≥ 90 days for incident scope |
| Encryption & access control | Preserved media stored encrypted (KMS/Key Vault), access-restricted |

---

## 6. Access for Forensics (¶(f))

Provide DoD access to additional information or equipment necessary for forensic analysis.

| Capability | Implementation |
|------------|----------------|
| Exportable evidence | Audit-log and SIEM exports; snapshot/image retrieval procedures (runbook) |
| Read access | Role-scoped read access to audit data for authorized investigators |

---

## 7. Cyber Incident Damage Assessment (¶(g))

Support damage-assessment activities if requested.

| Capability | Implementation |
|------------|----------------|
| Scope reconstruction | Immutable audit trail enables reconstruction of accessed records and actions |
| Data lineage | Record numbers + before/after hashes establish what changed and when |

---

## 8. Subcontractor Flow-Down (¶(m))

The clause must flow down to subcontractors handling CDI.

| Capability | Implementation |
|------------|----------------|
| Supplier records | Supplier Quality module + Document Control retain flow-down acknowledgements and supplier compliance evidence |
| Scoped access | Supplier (external) users are tightly scoped and never see other parties' CDI |

> Subcontractors must report incidents to the prime within 72 hours and to DoD via DIBNet; Sentinel QMS
> can hold the supplier compliance attestations and SCAR records supporting oversight.

## 10. Organizational Responsibilities

- Obtain a DoD-approved medium assurance certificate; register/report via DIBNet.
- Maintain the SSP and POA&M; perform the SPRS self-assessment score submission.
- Execute incident reporting within 72 hours and DC3 malware submission.
- Flow the clause down and oversee subcontractor compliance.
